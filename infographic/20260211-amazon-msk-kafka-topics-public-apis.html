Amazon MSK

新しいパブリック API による Kafka トピック管理の簡素化

CreateTopic, UpdateTopic, DeleteTopic API で Kafka AdminClient 不要のトピック管理を実現

2026 年 2 月 11 日

概要

Amazon MSK に 3 つの新しい API (CreateTopic、UpdateTopic、DeleteTopic) が追加され、MSK Provisioned クラスターの Kafka トピックを Kafka 管理クライアントのセットアップなしで管理できるようになりました。AWS CLI、AWS SDK、AWS CloudFormation などの使い慣れたインターフェースを使用して、プログラムによるトピックの作成、更新、削除が可能です。

既存の ListTopics および DescribeTopic API も Amazon MSK コンソールに統合され、すべてのトピック操作を 1 つの画面で実行できるようになりました。ガイド付きデフォルト値を使用した数クリックでのトピック作成・更新に加え、トピック設定、パーティションレベルの情報、メトリクスの包括的な可視化が提供されます。

トピック操作 API 一覧

📋 CreateTopic
新しいトピックを作成。トピック名、パーティション数、レプリケーションファクター、トピック設定を指定。
必要な権限: kafka-cluster:Connect kafka-cluster:CreateTopic
🔄 UpdateTopic
既存トピックのパーティション数またはトピック設定を更新。1 回のコールでどちらか一方のみ。
必要な権限: kafka-cluster:Connect kafka-cluster:DescribeTopic kafka-cluster:AlterTopic kafka-cluster:AlterTopicDynamicConfiguration
🗑️ DeleteTopic
トピックのデータ、メタデータ、パーティション情報を完全に削除。この操作は元に戻せない。
必要な権限: kafka-cluster:Connect kafka-cluster:DescribeTopic kafka-cluster:DeleteTopic
📃 ListTopics / 🔍 DescribeTopic
トピック一覧の取得やトピック詳細の参照。コンソールにも統合済み。
必要な権限: kafka-cluster:Connect kafka-cluster:DescribeTopic kafka-cluster:DescribeTopicDynamicConfiguration

設定方法

1 IAM ポリシーの設定
トピック操作に必要な IAM 権限を含むポリシーを作成します。kafka-cluster:Connect にはクラスター ARN を、その他のアクションにはトピック ARN を指定します。
JSON IAM ポリシー例 
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "kafka-cluster:Connect",
                "kafka-cluster:CreateTopic",
                "kafka-cluster:DescribeTopic",
                "kafka-cluster:AlterTopic",
                "kafka-cluster:DeleteTopic",
                "kafka-cluster:AlterTopicDynamicConfiguration",
                "kafka-cluster:DescribeTopicDynamicConfiguration"
            ],
            "Resource": [
                "arn:aws:kafka:us-east-1:123456789012:cluster/MyCluster/*",
                "arn:aws:kafka:us-east-1:123456789012:topic/MyCluster/*"
            ]
        }
    ]
}
2 トピックの作成
指定したクラスターに新しいトピックを作成します。
Bash AWS CLI - トピック作成 
# トピックを作成 (パーティション数 3、レプリケーションファクター 3)
aws kafka create-topic \
  --cluster-arn arn:aws:kafka:us-east-1:123456789012:cluster/MyCluster/abcd1234-abcd-dcba-4321-a1b2abcd9f9f-2 \
  --topic-name MyTopic \
  --partition-count 3 \
  --replication-factor 3
JSON レスポンス例 
{
    "topicArn": "arn:aws:kafka:us-east-1:123456789012:topic/MyCluster/.../MyTopic",
    "topicName": "MyTopic",
    "status": "CREATING"
}
3 トピックの更新
既存トピックのパーティション数を更新します。パーティション数とトピック設定は同時に更新できないため、別々の API コールで実行する必要があります。
Bash AWS CLI - パーティション数を 6 に更新 
aws kafka update-topic \
  --cluster-arn arn:aws:kafka:us-east-1:123456789012:cluster/MyCluster/abcd1234-abcd-dcba-4321-a1b2abcd9f9f-2 \
  --topic-name MyTopic \
  --partition-count 6
4 トピックの削除
トピックのデータ、メタデータ、パーティション情報を完全に削除します。この操作は元に戻せません。
Bash AWS CLI - トピック削除 
aws kafka delete-topic \
  --cluster-arn arn:aws:kafka:us-east-1:123456789012:cluster/MyCluster/abcd1234-abcd-dcba-4321-a1b2abcd9f9f-2 \
  --topic-name MyTopic

メリット

💼 ビジネス面

  • 運用効率の向上: Kafka AdminClient のセットアップや保守が不要になり、運用負荷が軽減
  • 追加コストなし: すべての MSK Provisioned クラスター (Kafka 3.6+) で追加料金なしで利用可能
  • Infrastructure as Code: CloudFormation でトピックを定義でき、インフラの一元管理を実現

🛠️ 技術面

  • 統一されたインターフェース: AWS CLI、SDK、コンソール、CloudFormation でトピックを管理
  • ガイド付きデフォルト値: コンソールから最適なデフォルト値でトピックを簡単に作成
  • 包括的な可視化: トピック設定、パーティション情報、メトリクスを一元的に確認可能

制約事項・注意点

制限事項
  • MSK Provisioned クラスターのみ対応 (MSK Serverless は非対応)
  • Apache Kafka バージョン 3.6.0 以降が必要
  • クラスターが ACTIVE 状態でなければ API を使用できない
  • API レスポンスのデータ更新は約 1 分間隔で反映される
考慮すべき点
  • UpdateTopic でパーティション数とトピック設定を同時に更新することはできない
  • DeleteTopic はデータ、メタデータ、パーティション情報を完全に削除し、元に戻せない
  • 適切な IAM 権限の設定が必要で、API ごとに異なる権限が求められる
  • ListTopics ではワイルドカードを使用してクラスター上のすべてのトピックをマッチできる

ユースケース

📝⚡
CloudFormation によるトピック管理の自動化
インフラストラクチャのデプロイ時に Kafka トピックも含めて一元的に管理したい場合。
Bash CloudFormation テンプレートでトピックをデプロイ 
aws cloudformation deploy \
  --template-file msk-topics.yaml \
  --stack-name msk-topics-stack
Kafka トピックを他の AWS リソースと同じ CloudFormation テンプレートで管理でき、環境のプロビジョニングと廃止を自動化
🚀🔄
CI/CD パイプラインでのトピック管理
アプリケーションのデプロイプロセスの一部として、必要な Kafka トピックを自動的に作成・更新したい場合。
Bash デプロイスクリプトでのトピック管理 
# 新しいトピックを作成
aws kafka create-topic \
  --cluster-arn $MSK_CLUSTER_ARN \
  --topic-name orders-v2 \
  --partition-count 12 \
  --replication-factor 3

# 既存トピックのパーティション数をスケールアウト
aws kafka update-topic \
  --cluster-arn $MSK_CLUSTER_ARN \
  --topic-name orders \
  --partition-count 24
Kafka AdminClient をセットアップすることなく、AWS CLI や SDK でデプロイパイプラインにトピック管理を組み込み可能

MSK コンソールによる運用管理
開発・運用チームがトピックの状態を確認し、必要に応じて設定変更を行いたい場合。

料金

追加料金なし
トピック管理 API は、すべての MSK Provisioned クラスター (Kafka 3.6+) で追加料金なしで利用可能です。MSK クラスター自体の料金については Amazon MSK 料金ページを参照してください。
