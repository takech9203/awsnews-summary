📋 概要

Amazon Bedrock が、フロンティアレベルの推論とエージェンティックコーディングをカバーする 6 つの新しいオープンウェイトモデルのサポートを開始しました。追加されたモデルは DeepSeek V3.2、MiniMax M2.1、GLM 4.7、GLM 4.7 Flash、Kimi K2.5、Qwen3 Coder Next です。

これらのモデルは、従来のプロプライエタリモデルと比較して大幅に低い推論コストでフロンティアクラスのパフォーマンスを提供します。新しい分散推論エンジン Project Mantle によって駆動され、OpenAI API 仕様との互換性を標準で備えています。

Before
  • オープンウェイトモデルの選択肢が限定的
  • フロンティアレベルの推論には高コストのプロプライエタリモデルに依存
  • エージェンティックコーディング向けモデルが不足
  • OpenAI SDK からの移行に追加の開発作業が必要
After
  • 6 つの新モデルで推論・コーディング・コスト効率を選択可能
  • フロンティアクラスの性能を大幅に低いコストで利用可能
  • Project Mantle による高パフォーマンスなサーバーレス推論
  • OpenAI API 互換でベース URL と API キーの変更のみで移行完了

⚙️ 設定方法

前提条件:

  • ✓ AWS アカウントを持ち、Amazon Bedrock へのアクセス権限が設定されていること
  • ✓ 使用するモデルへのアクセスが有効になっていること
  • ✓ OpenAI SDK を使用する場合は、Amazon Bedrock API キーが設定されていること

ステップ 1: モデルアクセスの有効化

AWS マネジメントコンソール > Amazon Bedrock > Model access から使用したいモデルを選択してアクセスを有効化します。

ステップ 2: 環境変数の設定

Bash OpenAI API 互換エンドポイントの設定 
# Amazon Bedrock API キーを設定
export OPENAI_API_KEY="your-bedrock-api-key"

# Amazon Bedrock エンドポイントを設定
export OPENAI_BASE_URL="https://bedrock-mantle.us-east-1.api.aws/v1"

ステップ 3: OpenAI SDK での利用

Python Chat Completions API でのリクエスト例 
from openai import OpenAI

client = OpenAI()

# Chat Completions API でのリクエスト例
completion = client.chat.completions.create(
    model="deepseek.deepseek-v3-2",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Hello!"}
    ]
)
print(completion.choices[0].message)

ステップ 4: ストリーミングレスポンス

Python ストリーミングレスポンスの利用 
from openai import OpenAI

client = OpenAI()

stream = client.chat.completions.create(
    model="deepseek.deepseek-v3-2",
    messages=[{"role": "user", "content": "Tell me a story"}],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        print(chunk.choices[0].delta.content, end="")

⚠️ デメリット・制約事項

🚧 制限事項

  • bedrock-mantle エンドポイントを通じて提供されるため、従来の Bedrock API エンドポイントとは異なるエンドポイントが必要
  • 各モデルの利用可能リージョンはモデルによって異なる場合がある
  • オープンウェイトモデルはプロプライエタリモデルと比較して、特定のベンチマークにおいて性能差がある場合がある

💡 考慮すべき点

  • Project Mantle エンドポイントを使用するには、Amazon Bedrock API キーの取得が必要
  • モデル選択にあたっては、各モデルの特化領域とユースケースの適合性を評価する必要がある
  • 料金はリージョンによって異なるため、デプロイ先のリージョンの料金を事前に確認することが推奨される

📝 まとめ

Amazon Bedrock に 6 つのオープンウェイトモデルが追加されたことで、エンタープライズ AI ワークロードのあらゆる領域をカバーするモデル選択肢が大幅に拡充されました。推論・エージェンティック特化、自律コーディング特化、コスト効率特化の 3 つのカテゴリにわたるモデル群により、ユースケースに最適なモデルを選択できます。Project Mantle 分散推論エンジンによる高パフォーマンスなサーバーレス推論と OpenAI API 互換エンドポイントにより、既存の OpenAI SDK ベースのアプリケーションからの移行も容易です。フロンティアクラスのパフォーマンスを低コストで利用でき、AI ワークロードのコスト最適化とパフォーマンス向上を同時に実現できるアップデートです。
