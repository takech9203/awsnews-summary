📋 概要

Amazon EC2 R7gd インスタンスが南米 サンパウロ sa-east-1 リージョンで利用可能になりました。R7gd は AWS Graviton3 プロセッサーを搭載し、DDR5 メモリと最大 3.8 TB のローカル NVMe SSD ストレージを提供するメモリ最適化インスタンスです。AWS Nitro System 上に構築され、オープンソース DB、インメモリキャッシュ、リアルタイムビッグデータ分析などに最適です。

❌ Before
•サンパウロで R7gd が利用不可、Graviton3 + ローカル NVMe SSD のメモリ最適化を活用できず
•ローカル SSD 付きメモリ最適化は旧世代インスタンスに依存
•最新 Graviton による価格パフォーマンス向上を南米で享受できず
✅ After
•サンパウロで R7gd を利用可能、Graviton3 + NVMe SSD を活用
•Graviton2 比で最大 25% コンピューティング向上、45% NVMe 向上
•DDR5 メモリで DDR4 比 50% 高いメモリ帯域幅を実現

🔧 主要機能

🚀 AWS Graviton3 プロセッサー搭載
  • ➔ Arm ベースの AWS 独自設計プロセッサー
  • ➔ Graviton2 と比較して最大 25% 優れたコンピューティングパフォーマンス
  • ➔ 最大 2 倍の浮動小数点パフォーマンス
  • ➔ 最大 2 倍の暗号化ワークロードパフォーマンス
  • ➔ ML ワークロードで最大 3 倍のパフォーマンス (bfloat16 サポート)
💾 ローカル NVMe SSD ストレージ
  • ➔ 最大 3.8 TB (2 x 1900 GB) のローカル NVMe SSD
  • ➔ Graviton2 比で最大 45% 優れたリアルタイム NVMe ストレージパフォーマンス
  • ➔ 高速かつ低レイテンシーのローカルストレージアクセス
🧠 DDR5 メモリと AWS Nitro System
  • ➔ DDR5 メモリにより DDR4 比で 50% 高いメモリ帯域幅
  • ➔ AWS Nitro System 上に構築、高い仮想化効率とセキュリティ
  • ➔ 16xlarge / metal サイズで EFA サポート

⚙️ 設定方法

前提条件

  • ✔ AWS アカウントと適切な IAM 権限
  • ✔ 南米 サンパウロ リージョン sa-east-1 へのアクセス
  • ✔ Arm (Graviton) アーキテクチャ対応の AMI

Step 1: 利用可能な R7gd インスタンスタイプを確認

Bash R7gd インスタンスタイプの確認 
# sa-east-1 リージョンで利用可能な R7gd インスタンスタイプを確認
aws ec2 describe-instance-types \
  --filters "Name=instance-type,Values=r7gd.*" \
  --region sa-east-1 \
  --query "InstanceTypes[].{Type:InstanceType,vCPU:VCpuInfo.DefaultVCpus,Memory:MemoryInfo.SizeInMiB,Storage:InstanceStorageInfo.TotalSizeInGB}" \
  --output table

Step 2: Graviton 対応 AMI を選択

Bash Amazon Linux 2023 Arm64 AMI を検索 
# Amazon Linux 2023 の Arm64 AMI を検索
aws ec2 describe-images \
  --owners amazon \
  --filters "Name=name,Values=al2023-ami-2023*-arm64" \
             "Name=state,Values=available" \
  --region sa-east-1 \
  --query "Images | sort_by(@, &CreationDate) | [-1].{ImageId:ImageId,Name:Name}" \
  --output table

Step 3: R7gd インスタンスを起動

Bash サンパウロリージョンで起動 
# R7gd インスタンスを起動
aws ec2 run-instances \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --instance-type r7gd.2xlarge \
  --region sa-east-1 \
  --subnet-id subnet-xxxxxxxxxxxxxxxxx \
  --security-group-ids sg-xxxxxxxxxxxxxxxxx \
  --key-name my-key-pair

⚠️ ローカル NVMe SSD は自動的にアタッチされますが、使用前にフォーマットとマウントが必要です。

✅ メリット

💼 ビジネス面

  • ✔ コスト効率の向上: Graviton3 により x86 比で優れた価格パフォーマンス、メモリ集約的ワークロードのコスト削減
  • ✔ レイテンシー最適化: 南米のエンドユーザーに近い場所でワークロードを実行し、応答時間を短縮
  • ✔ データレジデンシー対応: ブラジル LGPD などのデータ保護規制に準拠しながら最新インスタンスを活用

🔧 技術面

  • ✔ 高速ローカルストレージ: NVMe SSD による低レイテンシーアクセスで一時データ処理やキャッシュの性能向上
  • ✔ メモリ帯域幅の向上: DDR5 で DDR4 比 50% 高い帯域幅、メモリ集約的アプリのスループット向上
  • ✔ NVMe パフォーマンス: Graviton2 比で最大 45% 優れたリアルタイムストレージ性能
  • ✔ EFA サポート: 16xlarge / metal で Elastic Fabric Adapter 対応、HPC やノード間通信を最適化

⚠️ 制限事項・注意点

  • ⚠️ ローカル NVMe SSD はエフェメラルストレージ — インスタンス停止・終了でデータ消失
  • ⚠️ Arm (Graviton) アーキテクチャ対応のソフトウェア / AMI が必要 — x86 専用アプリはそのまま動作不可
  • ⚠️ すべてのリージョンで利用可能ではなく、対応リージョンは順次拡大中
  • 💡 永続データは Amazon EBS または Amazon S3 に別途保存する設計が必要
  • 💡 x86 からの移行時は Arm アーキテクチャへの互換性テストとパフォーマンス検証を推奨
  • 💡 ローカルストレージ容量はインスタンスサイズに固定、EBS のように柔軟なサイズ変更は不可

🎯 ユースケース

🗄️ オープンソースデータベースのキャッシュ層
南米向け E コマースプラットフォームが Redis / Memcached でデータベースクエリ結果をキャッシュし、応答時間を短縮。r7gd.4xlarge を使用。
📈 128 GiB メモリと 950 GB NVMe SSD で大量のキャッシュデータを低レイテンシー処理。DDR5 の高帯域幅によりキャッシュヒット率向上とレスポンス時間短縮を実現
📊 リアルタイムビッグデータ分析
金融テクノロジー企業が南米市場のトランザクションデータをリアルタイム分析し、不正検知を実行。r7gd.8xlarge x 3 ノードを使用。
📈 256 GiB メモリと 1.9 TB NVMe SSD を各ノードで活用。シャッフル / スピルオーバーデータをローカルストレージに高速書き込みし、分析パイプラインのスループットを向上
📁 一時データ処理パイプライン
メディア企業が動画トランスコーディングの一時ファイルを高速処理。r7gd.12xlarge を使用。
📈 384 GiB メモリと 2.85 TB (2 x 1425 GB) NVMe SSD で大容量一時ファイルを高速読み書き、トランスコーディング処理時間を大幅短縮
