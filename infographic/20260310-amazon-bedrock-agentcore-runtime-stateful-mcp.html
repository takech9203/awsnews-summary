📚 概要

Amazon Bedrock AgentCore Runtime がステートフルな Model Context Protocol (MCP) サーバー機能をサポート。開発者はエリシテーション、サンプリング、プログレス通知を活用した MCP サーバーを構築でき、既存のリソース、プロンプト、ツールのサポートに加えて、より高度なインタラクティブエージェントワークフローが実現可能に。

各ユーザーセッションは専用の microVM 上で分離されたリソースとともに実行され、Mcp-Session-Id ヘッダーを使用して複数のインタラクションにわたるセッションコンテキストを維持する。

🔄 Before / After

❌ Before
  • MCP サーバーはステートレスな動作に限定、セッション間でコンテキスト維持不可
  • ツール実行中にユーザーからインタラクティブに入力を収集する手段がなかった
  • サーバーからクライアント側の LLM にテキスト生成をリクエストする仕組みがなかった
  • 長時間オペレーションの進捗をリアルタイムで通知する方法がなかった
✅ After
  • エリシテーションにより、サーバー起動のマルチターン会話でユーザー情報を収集可能に
  • サンプリングにより、サーバーがクライアントに AI テキスト生成をリクエスト可能に
  • プログレス通知により、長時間オペレーションの進捗をリアルタイム通知可能に
  • 専用 microVM による分離とセッション ID によるコンテキスト維持が可能に

🔑 主要機能

💬
Elicitation
サーバーからクライアントへマルチターン会話を開始し、ユーザーの嗜好や設定情報をインタラクティブに収集。旅行予約エージェントが座席の好みや食事制限を確認するなどの場面で活用。
🤖
Sampling
MCP サーバーがクライアント側の LLM に AI テキスト生成をリクエスト。パーソナライズされた推薦文やコンテンツ生成に活用可能。サーバー側で別途モデル管理が不要。
🕐
Progress Notifications
長時間実行されるオペレーションの進捗状況をリアルタイムでクライアントに通知。フライト検索、予約処理、データ分析などの処理中にユーザーへフィードバックを提供。
🛡
Stateful Session Management
各ユーザーセッションが専用 microVM で実行されリソースが分離。Mcp-Session-Id ヘッダーにより複数インタラクションにわたるコンテキストを維持。

💻 実装例

🔨 MCP サーバーの実装

Python Stateful MCP Server - Elicitation / Sampling / Progress 
from mcp.server import Server

server = Server("my-stateful-server")

@server.tool()
async def book_flight(destination: str):
    # Elicitation: ユーザーの座席の好みを収集
    seat_pref = await server.elicit(
        prompt="座席のご希望はありますか?",
        options=["窓側", "通路側", "指定なし"]
    )

    # Progress: 検索中であることを通知
    await server.notify_progress(0.3, "フライトを検索中...")

    # Sampling: クライアント LLM にパーソナライズされた提案を生成依頼
    recommendation = await server.sample(
        prompt=f"{destination} へのフライト提案を作成してください"
    )

    await server.notify_progress(1.0, "検索完了")
    return recommendation

client = boto3.client('bedrock-agentcore-control')

response = client.create_agent_runtime(
    agentRuntimeName='my-stateful-mcp-server',
    agentRuntimeArtifact={
        'containerConfiguration': {
            'containerUri': '123456789012.dkr.ecr.us-east-1.amazonaws.com/my-mcp-server:latest'
        }
    },
    roleArn='arn:aws:iam::123456789012:role/AgentCoreRuntimeRole',
    protocolConfiguration={
        'serverProtocol': 'MCP'
    },
    lifecycleConfiguration={
        'idleRuntimeSessionTimeout': 300,
        'maxLifetime': 3600
    },
    networkConfiguration={
        'networkMode': 'PUBLIC'
    }
)

# セッションの開始とツール呼び出し
# レスポンスの Mcp-Session-Id ヘッダーを以降のリクエストに含める

  -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: <session-id>" \
  -d '{"tool": "book_flight", "arguments": {"destination": "Tokyo"}}'

🎯 ユースケース

✈ ユースケース 1: 旅行予約エージェント
ユーザーが旅行の予約を行う際に、エージェントがインタラクティブに詳細を確認しながら最適なプランを提案。エリシテーションで旅行日程や予算を確認し、サンプリングでパーソナライズされた旅程を生成。ユーザーとの対話を通じて要件を正確に把握し、最適な旅行プランを提供。
💡 ユースケース 2: カスタマーサポートエージェント
技術サポートにおいて、問題の切り分けのために段階的にユーザーから情報を収集し、解決策を提示。エリシテーションで環境情報やエラーメッセージを収集し、ナレッジベースを検索後、サンプリングで的確な解決策を生成。セッションコンテキストを維持しながら段階的に問題を特定。
📈 ユースケース 3: データ分析パイプライン
大量のデータ分析処理をバックグラウンドで実行しながら、プログレス通知で進捗状況をリアルタイムでユーザーに通知。データ読み込み、前処理、分析実行の各段階で進捗を報告し、完了後にサンプリングで分析結果の要約を生成。

🔬 メリットと注意点

🌟 メリット

  • エリシテーションとプログレス通知により、エージェントとのインタラクションがより自然で情報量の多いものに
  • サンプリング機能により、クライアント側の LLM を活用したパーソナライズされたコンテンツ生成が可能
  • マルチターン会話やコンテキスト維持により、予約やサポートなどの複雑なビジネスプロセスを自動化
  • 各セッションが専用 microVM で動作し、マルチテナント環境でのセキュリティが確保
  • Mcp-Session-Id による標準的なセッション管理で独自実装が不要
  • MCP 標準準拠で既存エコシステムとの互換性を維持

⚠ 制限事項・考慮点

  • 専用 microVM 使用のため、ステートレスモードと比較してリソースコストが増加する可能性
  • アイドルタイムアウトと最大ライフタイムの制限があり、長時間セッション維持には設計上の考慮が必要
  • エリシテーションやサンプリングはクライアント側の対応が必要で、すべての MCP クライアントが対応しているとは限らない
  • microVM 起動時間がコールドスタートとして影響する可能性
  • セッション数に応じた microVM のスケーリングとコスト管理の計画が必要

💰 料金

AgentCore Runtime の料金は、microVM の実行時間とリソース使用量に基づく。ステートフルセッションでは専用 microVM が割り当てられるため、セッション数とセッション期間が料金に影響する。詳細な料金情報は Amazon Bedrock の料金ページを参照のこと。
