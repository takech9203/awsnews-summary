Amazon SNS SMS 送信 2 Regions NEW

Amazon SNS
SMS 送信の追加リージョンサポート

2026 年 2 月 19 日

概要

Amazon SNS を Asia Pacific (New Zealand) および Asia Pacific (Taipei) リージョンで使用しているお客様が、200 を超える国と地域のサブスクライバに SMS メッセージを送信できるようになりました。SMS メッセージは AWS End User Messaging を介して送信されます。Amazon SNS は現在 32 の AWS リージョンで SMS 送信をサポートしています。

Before - アップデート前の課題
  • Asia Pacific (New Zealand) および Asia Pacific (Taipei) リージョンでは SNS からの SMS 送信が利用できなかった
  • これらのリージョンのお客様は SMS 送信のために他のリージョンを使用する必要があった
After - アップデート後の改善
  • 2 つの新しいリージョンから 200 を超える国と地域への SMS 送信が可能になった
  • SNS の SMS 送信対応リージョンが 32 に拡大した

主要機能

🌍
追加リージョン対応
Asia Pacific (New Zealand) と Asia Pacific (Taipei) の 2 つのリージョンで SMS 送信が利用可能に。
  • Asia Pacific (New Zealand) リージョンでの SMS 送信サポート
  • Asia Pacific (Taipei) リージョンでの SMS 送信サポート
  • 200 を超える国と地域への送信が可能
🔗
AWS End User Messaging 統合
SMS メッセージは AWS End User Messaging を介して送信され、既存の SNS pub/sub メッセージング機能と統合。
  • AWS End User Messaging 経由での SMS 配信
  • 既存の SNS pub/sub モデルを活用
  • 既存ワークフローとのシームレスな連携

設定方法

前提条件
  • 対応リージョンで Amazon SNS が有効であること
  • SMS 送信に必要な IAM 権限が設定されていること
Step 1: SNS トピックまたは電話番号サブスクリプションの設定

対応リージョンで SNS トピックを作成し、SMS 送信先の電話番号をサブスクライバとして追加します。

Bash SNS トピック作成と SMS サブスクリプション追加 
# New Zealand リージョンで SNS トピックを作成
aws sns create-topic \
  --name my-sms-topic \
  --region ap-southeast-5

# 電話番号をサブスクライバとして追加
aws sns subscribe \
  --topic-arn arn:aws:sns:ap-southeast-5:123456789012:my-sms-topic \
  --protocol sms \
  --notification-endpoint "+8190XXXXXXXX" \
  --region ap-southeast-5

# SMS メッセージを送信
aws sns publish \
  --topic-arn arn:aws:sns:ap-southeast-5:123456789012:my-sms-topic \
  --message "Hello from New Zealand region!" \
  --region ap-southeast-5

New Zealand リージョンで SNS トピックを作成し、電話番号のサブスクリプションを追加してメッセージを送信する例です。

ユースケース

📱 ニュージーランドでの通知サービス
ニュージーランドに拠点を持つ企業が、顧客向けの SMS 通知 (注文確認、配送状況、アラートなど) をローカルリージョンから直接送信したい。
Asia Pacific (New Zealand) リージョンから直接 SMS を送信することで、データ主権の確保と低レイテンシーを実現。他のリージョンへの依存を排除し、運用の簡素化が可能。
📊 台湾市場向けのマーケティング
台湾市場で事業を展開する企業が、プロモーションや二要素認証 (2FA) のための SMS を効率的に配信したい。
Asia Pacific (Taipei) リージョンから SMS を送信することで、台湾の顧客へのレスポンス時間を短縮。SNS の pub/sub モデルを活用してスケーラブルな SMS 配信基盤を構築可能。
🔒 IoT デバイスアラート
オセアニア・東アジア地域に展開する IoT システムで、デバイスの異常検知時に管理者へ即時に SMS アラートを送信したい。
最寄りリージョンからの SMS 送信によりアラートの到達時間を短縮。SNS トピックと組み合わせて、複数の管理者へ同時にアラートを配信可能。

メリット

ビジネス面
🛡️
データ主権の確保
New Zealand と Taipei のお客様がローカルリージョンから SMS を送信できるため、データレジデンシー要件への準拠が容易に。
⚡
レイテンシの改善
最寄りリージョンからの送信により、API レスポンス時間が短縮され、SMS 配信のスピードが向上。
技術面
🌍
リージョン選択の柔軟性
32 リージョンから最適な送信元を選択可能。ワークロードの配置やデータ要件に応じた柔軟な構成が実現。
🔧
既存アーキテクチャとの統合
SNS の pub/sub モデルを活用した SMS 配信により、既存のメッセージングアーキテクチャとシームレスに統合。

制約事項・考慮点

⚠ 制限事項
  • 送信先の国や地域によって送信料金が異なる
  • 一部の国では SMS 送信に事前登録が必要な場合がある
ℹ 考慮すべき点
  • SMS 送信にはお客様が対象国の規制に準拠する必要がある
  • 送信先番号の形式や規制は国ごとに異なるため、事前に対応国・地域一覧を確認することを推奨

料金

SMS 送信料金はリージョンと送信先の国により異なります。詳細は Amazon SNS の料金ページをご確認ください。
