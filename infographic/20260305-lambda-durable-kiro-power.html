Kiro IDE の Powers メニューまたは Kiro Powers ページから「Lambda Durable Functions」Power をワンクリックインストール。

Step 2: プロジェクト開始
Bash SDK インストール

npm install @aws-lambda/durable-functions
Step 3: デプロイ
Bash SAM デプロイ 
sam build
sam deploy --guided

💡 ユースケース

1. 注文処理パイプライン

バリデーション、決済認証、在庫確保、配送手配を順序立てて実行し、障害時は自動ロールバック。

TypeScript注文処理
export const handler = durable(async (ctx, event) => {
  const validated = await ctx.step('validate',
    () => validateOrder(event.order));
  const payment = await ctx.step('authorize',
    () => authorizePayment(validated));
  const inventory = await ctx.step('allocate',
    () => allocateInventory(validated));
  await ctx.step('fulfill',
    () => fulfillOrder(payment, inventory));
});

効果: リトライ戦略や補償トランザクションを AI が提案し、堅牢なパイプラインを迅速構築

2. Human-in-the-Loop AI オーケストレーション

AI が文書分析後、人間の承認を待ってからアクション実行するワークフロー。

TypeScript承認待ちフロー
export const handler = durable(async (ctx, event) => {
  const analysis = await ctx.step('analyze',
    () => analyzeDocument(event.document));
  const approval = await ctx.wait('human-approval', {
    type: 'event',
    event: 'approval-received'
  });
  if (approval.approved) {
    await ctx.step('execute',
      () => executeAction(analysis));
  }
});

効果: Wait 中は課金なし。承認待ちコストを抑えつつワークフロー実装

3. 決済連携ワークフロー

複数決済プロバイダーにまたがる認証、不正検知、決済処理をトランザクション状態維持で実行。

TypeScript決済処理
export const handler = durable(async (ctx, event) => {
  const fraud = await ctx.step('fraud-check',
    () => checkFraud(event.payment));
  const auth = await ctx.step('authorize',
    () => authorizeWithProvider(event.payment));
  const settlement = await ctx.step('settle',
    () => settlePayment(auth));
  await ctx.step('notify',
    () => notifyCompletion(settlement));
});

効果: 障害時もトランザクション状態自動保存、リプレイで中断箇所から再開
