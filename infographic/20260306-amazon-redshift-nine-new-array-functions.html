Amazon Redshift

9 つの新しい配列関数で半構造化データ処理を強化

SUPER データ型向け - 検索、集合演算、変換を SQL 内で直接実行
2026 年 3 月 6 日

📝 概要

Amazon Redshift が SUPER データ型に格納された半構造化データを操作するための 9 つの新しい配列関数 をサポート。配列の検索、比較、ソート、変換を SQL クエリ内で直接実行できるようになった。

ARRAY_CONTAINS
ARRAY_POSITION
ARRAY_POSITIONS
ARRAY_UNION
ARRAY_EXCEPT
ARRAY_INTERSECTION
ARRAYS_OVERLAP
ARRAY_SORT
ARRAY_DISTINCT

🔄 Before / After

Before
❌要素検索に複雑な SQL やカスタム UDF が必要
❌集合演算はアプリケーション側で処理
❌UNNEST と再集約の冗長なクエリが必須
→
After
✅関数一発で配列内の要素検索が可能
✅和集合・差集合・積集合を SQL で直接実行
✅ソート・重複排除もネイティブ関数で完結

🔧 3 つのカテゴリ

🔍 検索関数
📌ARRAY_CONTAINS - 値の存在判定
📌ARRAY_POSITION - 最初の出現位置
📌ARRAY_POSITIONS - 全出現位置のリスト
📐 集合演算関数
📌ARRAY_UNION - 和集合
📌ARRAY_EXCEPT - 差集合
📌ARRAY_INTERSECTION - 積集合
📌ARRAYS_OVERLAP - 共通要素の有無
🔄 変換関数
📌ARRAY_SORT - 昇順/降順ソート
📌ARRAY_DISTINCT - 重複排除

🛠️ 設定方法

前提条件: Redshift クラスタ稼働中、SUPER 型カラムのテーブルが存在すること。

Step 1: テーブル準備
SQL SUPER 型テーブル作成 
CREATE TABLE products (
    product_id INT,
    name VARCHAR(256),
    tags SUPER,
    categories SUPER
);
Step 2: 検索関数の使用
SQL ARRAY_CONTAINS 
SELECT name, tags
FROM products
WHERE ARRAY_CONTAINS(tags, 'sale');
Step 3: 集合演算の使用
SQL ARRAY_UNION / ARRAY_INTERSECTION 
SELECT name,
  ARRAY_UNION(tags, categories) AS all_labels,
  ARRAY_INTERSECTION(tags, categories)
    AS common_labels
FROM products;
Step 4: 変換関数の使用
SQL ARRAY_SORT + ARRAY_DISTINCT 
SELECT ARRAY_SORT(
  ARRAY_DISTINCT(
    ARRAY_UNION(tags, categories)
  )
) AS unique_sorted_labels
FROM products;

⚠️ デメリット・制約

🚫SUPER 型のカラムのみ対応。VARCHAR の JSON には直接適用不可
🔧ネストされた配列の動作は関数ごとに異なる可能性
📏非常に大きい配列ではパフォーマンスへの影響を考慮
💡既存 UNNEST ベースのクエリからの移行は手動対応が必要
💡既存テーブルは SUPER 型への JSON_PARSE 変換が事前に必要

💡 ユースケース

1. E コマースの商品タグ分析

商品に付与された複数タグを SUPER 型で管理し、タグベースの商品検索やカテゴリ分析を実行。

SQL
SELECT product_id, name
FROM products
WHERE ARRAY_CONTAINS(tags, 'sale')
  AND ARRAY_CONTAINS(tags, 'electronics');

効果: WHERE 句で直接タグフィルタリング、アプリ側ロジック不要

2. IoT センサーデータの集合演算

複数デバイスのアラート配列を比較し、共通する異常パターンを SQL 内で特定。

SQL
SELECT
  ARRAY_INTERSECTION(
    a.alerts, b.alerts
  ) AS common_alerts,
  ARRAY_EXCEPT(
    a.alerts, b.alerts
  ) AS unique_to_a
FROM device_readings a
JOIN device_readings b
  ON a.timestamp = b.timestamp;

効果: デバイス間の異常比較分析を SQL 内で完結

3. ユーザー行動データの正規化

クリックストリームやイベントログの配列を重複排除・ソートして行動パターンを分析。

SQL
SELECT user_id,
  ARRAY_SORT(
    ARRAY_DISTINCT(visited_pages)
  ) AS unique_pages_sorted
FROM user_sessions
WHERE session_date = '2026-03-06';

効果: レコメンデーションやファネル分析の前処理を効率化

🔗 関連サービス・機能

🗄️Redshift SUPER データ型: 配列関数の操作対象となるネイティブ半構造化データ型

📋 まとめ

Amazon Redshift に追加された 9 つの配列関数 により、SUPER データ型の半構造化データ操作が SQL 内で大幅に簡素化された。検索・集合演算・変換 の 3 カテゴリで頻出操作がネイティブ関数として利用可能になり、複雑な UDF やアプリケーション側処理が不要に。JSON や半構造化データを扱っている場合は、既存クエリを新しい配列関数に置き換えることで 可読性とパフォーマンスの向上 が期待できる。
