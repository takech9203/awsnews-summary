概要

AWS は 2026 年 3 月 13 日、Amazon MSK のプロビジョニングクラスターにおいて、Standard ブローカーで AWS Graviton3 ベースの M7g インスタンスがアフリカ (ケープタウン) リージョンで利用可能になったことを発表しました。

Graviton3 ベースの M7g インスタンスは、同等の M5 インスタンスと比較して最大 24% のコンピューティングコスト削減と最大 29% の読み書きスループット向上を実現します。アフリカリージョンで Apache Kafka ワークロードを運用するユーザーにとって、コスト効率とパフォーマンスの両面で大きなメリットをもたらすアップデートです。

Before / After

Before: M5 インスタンスのみ
  • アフリカ (ケープタウン) リージョンでは Standard ブローカーで Graviton3 ベースの M7g インスタンスが利用できなかった
  • M5 インスタンスに依存しており、最新世代の Graviton プロセッサーによるコスト削減とスループット向上を享受できなかった
  • 大量のストリーミングデータを処理する際にコストパフォーマンスが最適ではなかった
After: M7g インスタンスが選択可能
  • アフリカ (ケープタウン) リージョンで MSK Standard ブローカーに M7g インスタンスを選択できるようになった
  • M5 インスタンスと比較して最大 24% のコンピューティングコスト削減を実現
  • M5 インスタンスと比較して最大 29% 高い読み書きスループットを達成

設定方法

前提条件

  • AWS アカウントと Amazon MSK に対する適切な IAM 権限
  • アフリカ (ケープタウン) リージョン (af-south-1) が有効化されていること
  • MSK クラスター用の VPC、サブネット、セキュリティグループが設定済みであること

ステップ 1: 新規 MSK クラスターを M7g ブローカーで作成

Bash M7g インスタンスを使用した MSK クラスターの作成 
# M7g インスタンスを使用した MSK クラスターの作成
aws kafka create-cluster \
  --cluster-name my-msk-cluster \
  --broker-node-group-info '{
    "InstanceType": "kafka.m7g.xlarge",
    "ClientSubnets": ["subnet-xxx", "subnet-yyy", "subnet-zzz"],
    "SecurityGroups": ["sg-xxx"],
    "StorageInfo": {
      "EbsStorageInfo": {
        "VolumeSize": 100
      }
    }
  }' \
  --kafka-version "3.6.0" \
  --number-of-broker-nodes 3 \
  --region af-south-1

ステップ 2: 既存の M5 クラスターを M7g にアップグレード

Bash 既存クラスターのブローカータイプを M7g に更新 
# 既存クラスターのブローカータイプを M7g に更新
aws kafka update-broker-type \
  --cluster-arn arn:aws:kafka:af-south-1:123456789012:cluster/my-cluster/xxx-x \
  --current-version "K1XXXXXXXXXXXXXXXX" \
  --target-instance-type "kafka.m7g.xlarge" \
  --region af-south-1

ステップ 3: アップグレード状況の確認

Bash クラスターの状態を確認 
# クラスターの状態を確認
aws kafka describe-cluster \
  --cluster-arn arn:aws:kafka:af-south-1:123456789012:cluster/my-cluster/xxx-x \
  --region af-south-1 \
  --query "ClusterInfo.{State:State,BrokerType:BrokerNodeGroupInfo.InstanceType}"

メリット・デメリット

メリット

ビジネス面

  • M5 インスタンスから M7g への移行により最大 24% のコンピューティングコスト削減を実現
  • アフリカのエンドユーザーに近い場所で Kafka クラスターを運用し、レイテンシーを短縮
  • Graviton3 プロセッサーはエネルギー効率が高く、カーボンフットプリントの削減に貢献

技術面

  • M5 比で最大 29% 高い読み書きスループットにより、高トラフィックのストリーミングワークロードに対応
  • 既存の M5 クラスターをコンソールまたは CLI からインプレースアップグレード可能
  • Apache Kafka はプラットフォーム非依存のため、Arm アーキテクチャへの移行時にアプリケーション側の変更が不要

制約・注意点

制限事項

  • 本アップデートはアフリカ (ケープタウン) リージョンのみが対象
  • アフリカ (ケープタウン) はオプトインリージョンであり、利用前にアカウントで有効化が必要
  • Standard ブローカーが対象であり、Express ブローカーについては別途確認が必要

考慮すべき点

  • M5 から M7g へのアップグレード中はローリング方式でブローカーが再起動され、一時的にパフォーマンスが低下する可能性
  • アップグレード前に、本番環境と同等のテスト環境でパフォーマンスを検証することを推奨

ユースケース

ユースケース 1: アフリカ向けリアルタイムイベント処理
南アフリカの金融サービス企業が、取引イベントをリアルタイムで処理し、不正検知や取引分析を行いたい場合。M7g インスタンスにより高スループットかつ低コストな Kafka クラスターを構築できます。
Bash 高スループット MSK クラスターの作成 
# M7g ブローカーを使用した高スループット MSK クラスターの作成
aws kafka create-cluster \
  --cluster-name trading-events-cluster \
  --broker-node-group-info '{
    "InstanceType": "kafka.m7g.2xlarge",
    "ClientSubnets": ["subnet-aaa", "subnet-bbb", "subnet-ccc"],
    "SecurityGroups": ["sg-xxx"],
    "StorageInfo": {"EbsStorageInfo": {"VolumeSize": 500}}
  }' \
  --kafka-version "3.6.0" \
  --number-of-broker-nodes 3 \
  --region af-south-1
M7g インスタンスにより最大 29% 高いスループットを実現し、取引イベントの処理遅延を低減。同時に最大 24% のコスト削減でインフラコストを最適化
ユースケース 2: IoT データストリーミング
アフリカの通信事業者が、IoT デバイスからの大量のテレメトリデータを収集・処理するためのストリーミング基盤を構築したい場合。
Bash IoT データ収集用 MSK クラスター 
# IoT データ収集用 MSK クラスター
aws kafka create-cluster \
  --cluster-name iot-telemetry-cluster \
  --broker-node-group-info '{
    "InstanceType": "kafka.m7g.xlarge",
    "ClientSubnets": ["subnet-aaa", "subnet-bbb", "subnet-ccc"],
    "SecurityGroups": ["sg-xxx"],
    "StorageInfo": {"EbsStorageInfo": {"VolumeSize": 1000}}
  }' \
  --kafka-version "3.6.0" \
  --number-of-broker-nodes 3 \
  --region af-south-1
ケープタウンリージョンでの低レイテンシーなデータ収集と、Graviton3 による高いコスト効率を両立。大量の IoT デバイスからのデータストリームを効率的に処理
ユースケース 3: 既存 M5 クラスターのコスト最適化
既にケープタウンリージョンで M5 ベースの MSK クラスターを運用している企業が、コスト削減とパフォーマンス向上を目的にアップグレードしたい場合。
Bash 既存クラスターを M7g にアップグレード 
# 既存クラスターを M7g にアップグレード
aws kafka update-broker-type \
  --cluster-arn arn:aws:kafka:af-south-1:123456789012:cluster/existing-cluster/xxx-x \
  --current-version "K1XXXXXXXXXXXXXXXX" \
  --target-instance-type "kafka.m7g.xlarge" \
  --region af-south-1
アプリケーション側の変更なしにインプレースアップグレードが可能。最大 24% のコスト削減と最大 29% のスループット向上を即座に実現
