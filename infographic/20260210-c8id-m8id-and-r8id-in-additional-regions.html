概要

カスタム Intel Xeon 6 プロセッサを搭載した Amazon EC2 C8id、M8id、R8id インスタンスが、ヨーロッパ (フランクフルト)、アジアパシフィック (東京)、ヨーロッパ (スペイン) リージョンで新たに利用可能になりました。前世代の C6id/M6id/R6id と比較して最大 43% の性能向上と 3.3 倍のメモリ帯域幅を実現し、最大 384 vCPU、3TiB のメモリ、22.8TB の NVMe SSD ストレージを提供します。

Before - アップデート前の課題
  • C8id/M8id は米国リージョンのみで、東京やフランクフルトでは利用不可
  • R8id はフランクフルトで利用可能だが、東京やスペインでは利用不可
  • 東京リージョンのユーザーは最新世代の I/O 最適化インスタンスにアクセスできず、リージョン間通信が必要
After - アップデート後の改善
  • C8id/M8id/R8id 全 3 ファミリーが東京リージョンで利用可能に
  • フランクフルトで C8id/M8id 追加、ヨーロッパのデータローカリティ要件に対応
  • スペインで M8id/R8id 追加、南ヨーロッパの選択肢が拡大
  • 最大 43% 性能向上と 3.3 倍のメモリ帯域幅をより多くのリージョンで活用可能

新たに C8id/M8id/R8id が利用可能になった 3 リージョンと、各インスタンスファミリーが対応するワークロードの関係を示しています。

インスタンスファミリー比較

⚙ C8id - コンピュート最適化
高性能 Web サーバー、バッチ処理、分散分析、広告配信、動画エンコードなどコンピュート集約型ワークロードに最適。
  • 最大 384 vCPU
  • 最大 768 GiB メモリ
  • 最大 22.8 TB NVMe SSD
  • ゲームサーバーにも対応
⚡ M8id - 汎用
アプリケーションサーバー、マイクロサービス、エンタープライズアプリ、中小規模データベースに最適なバランス型。
  • 最大 384 vCPU
  • 最大 1.5 TiB メモリ
  • 最大 22.8 TB NVMe SSD
  • 最もリージョン展開が広い
🚀 R8id - メモリ最適化
インメモリデータベース、リアルタイムビッグデータ分析、大規模キャッシュ、科学計算向け。
  • 最大 384 vCPU
  • 最大 3 TiB メモリ
  • 最大 22.8 TB NVMe SSD
  • 最大容量のメモリ搭載

設定方法

前提条件
  • AWS アカウントと EC2 起動権限
  • 対象リージョン (東京、フランクフルト、スペイン) へのアクセス
  • 適切なサービスクォータの確認
Step 1: 利用可能なインスタンスタイプの確認
Bash C8id インスタンスタイプ確認 
# 東京リージョンで利用可能な C8id インスタンスタイプを確認
aws ec2 describe-instance-types \
  --filters "Name=instance-type,Values=c8id*" \
  --region ap-northeast-1 \
  --query "InstanceTypes[].{Type:InstanceType,vCPU:VCpuInfo.DefaultVCpus,Memory:MemoryInfo.SizeInMiB}" \
  --output table
Step 2: インスタンスの起動
Bash M8id インスタンス起動 
# 東京リージョンで M8id インスタンスを起動
aws ec2 run-instances \
  --instance-type m8id.24xlarge \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --subnet-id subnet-xxxxxxxxxxxxxxxxx \
  --security-group-ids sg-xxxxxxxxxxxxxxxxx \
  --key-name my-key-pair \
  --region ap-northeast-1
Step 3: Instance Bandwidth Configuration の確認
Bash 帯域幅サポート確認 
# 帯域幅の重み付け調整サポートを確認
aws ec2 describe-instance-types \
  --instance-types m8id.24xlarge \
  --query 'InstanceTypes[0].NetworkInfo.BandwidthWeightingSupport' \
  --region ap-northeast-1

ユースケース

🏧 東京リージョンでの高性能データベース
日本国内の金融機関が、大規模な OLTP データベースを低レイテンシーで運用し、I/O 集約型のトランザクション処理を高速化。R8id の大容量メモリ (最大 3TiB) と高速 NVMe SSD ストレージを活用。
✅ R8id により I/O 集約型データベースで最大 46% の性能向上。東京リージョンでの運用により国内ユーザーへのレイテンシーを最小化。
📊 フランクフルトリージョンでのリアルタイム分析
ヨーロッパの E コマース企業が、リアルタイムでユーザー行動データを分析し、パーソナライズされた推奨を提供。M8id の汎用的なバランスを活用。
✅ M8id により最大 30% 高速なクエリ結果を実現。GDPR 準拠のためのデータローカリティ要件にも対応。
🎮 東京リージョンでのゲームサーバー
オンラインゲーム運営企業が、日本国内のプレイヤー向けに高性能なゲームサーバーを低レイテンシーで提供。C8id のコンピュート最適化性能を活用。
✅ C8id の高速 I/O と Instance Bandwidth Configuration でネットワーク帯域幅を優先的に割り当て、多数の同時接続プレイヤーに対応。

制約事項・考慮点

⚠ 制限事項
  • インスタンスファミリーによって利用可能なリージョンが異なる (C8id はスペインリージョンでは未提供)
  • 前世代インスタンスからの移行には AMI 互換性の確認が必要
  • 一部の古い OS バージョンではドライバ更新が必要な場合がある
ℹ 考慮すべき点
  • 大容量 NVMe SSD はエフェメラルストレージのため、永続的なデータ保存には Amazon EBS との併用が必要
  • 大規模インスタンスは起動時間が長くなる可能性がある
  • 適切なインスタンスファミリーの選択にはワークロード特性の考慮が必要
  • スポットインスタンスの中断リスクを考慮した設計が必要
