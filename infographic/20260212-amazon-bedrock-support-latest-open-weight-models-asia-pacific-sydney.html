📋 概要

Amazon Bedrock が Asia Pacific (Sydney) リージョンで最新のオープンウェイトモデルのサポートを開始。bedrock-runtime エンドポイントと bedrock-mantle エンドポイントの両方を使用して、DeepSeek、Google、MiniMax、Mistral、Moonshot AI、Nvidia、OpenAI など 7 社 のプロバイダのモデルが利用可能になりました。

⚡ 主要機能

🔌
bedrock-runtime エンドポイント
InvokeModel、Converse、Chat Completions API を使用したリージョン固有の推論リクエスト用エンドポイント。標準的な Bedrock API として利用可能です。
🚀
bedrock-mantle エンドポイント
OpenAI 互換エンドポイントを使用した推論リクエスト用。Project Mantle ベースの分散推論エンジンによって駆動されます。
⚙️
Project Mantle
大規模 ML モデル提供のための分散推論エンジン。高パフォーマンスかつ信頼性の高いサーバーレス推論と OpenAI API 互換性を提供します。

ステップ 1: モデルアクセスの有効化

Amazon Bedrock コンソールで Asia Pacific (Sydney) リージョンを選択し、使用するオープンウェイトモデルへのアクセスをリクエストします。

ステップ 2: bedrock-runtime での呼び出し例

Python bedrock-runtime エンドポイントでの Converse API 使用例 
import boto3
import json

bedrock_runtime = boto3.client(
    'bedrock-runtime',
    region_name='ap-southeast-2'  # Sydney
)

# Converse API を使用した推論リクエスト
response = bedrock_runtime.converse(
    modelId='<model-id>',
    messages=[
        {
            'role': 'user',
            'content': [
                {'text': 'Hello, how are you?'}
            ]
        }
    ]
)

print(response['output']['message']['content'][0]['text'])

ステップ 3: bedrock-mantle での呼び出し例

Python bedrock-mantle エンドポイントでの OpenAI 互換 API 使用例 
from openai import OpenAI

# bedrock-mantle エンドポイントを使用
client = OpenAI(
    base_url='https://bedrock-mantle.ap-southeast-2.amazonaws.com/v1',
    api_key='<credentials>'
)

response = client.chat.completions.create(
    model='<model-id>',
    messages=[
        {'role': 'user', 'content': 'Hello, how are you?'}
    ]
)

print(response.choices[0].message.content)

💼 メリット

💼 ビジネス面

  • データ主権: Oceania 地域のデータ主権要件を満たしながらモデルを利用可能
  • 低レイテンシ: ローカルリージョンからの推論で応答時間が改善
  • 選択肢の拡大: 複数プロバイダの最新モデルから最適なものを選択可能

🛠️ 技術面

  • OpenAI 互換 API: bedrock-mantle による OpenAI API 互換性で既存コードの移行が容易
  • Project Mantle: 分散推論エンジンによる高パフォーマンスなサーバーレス推論
  • 複数エンドポイント: ユースケースに応じて最適なエンドポイントを選択可能

⚠️ 制約事項・注意点

制限事項
  • 一部のモデルはリージョンによって利用可能性が異なる場合がある
考慮すべき点
  • モデルごとの料金とクォータを確認すること
  • bedrock-runtime と bedrock-mantle のどちらのエンドポイントがユースケースに適しているかを事前に検討すること

🎯 ユースケース

🌎📊
Oceania 地域のデータ主権要件を満たす AI アプリケーション
オーストラリアの企業がデータ主権要件を満たしつつ、最新の AI モデルを活用したアプリケーションを構築したい。
Sydney リージョンでオープンウェイトモデルが利用可能になったことで、データを国外に送信することなく、複数プロバイダの最新モデルから最適なものを選択して低レイテンシで推論を実行できます。
🔄🤖
OpenAI SDK からの移行
既存の OpenAI API ベースのアプリケーションを AWS マネージド環境に移行したい。
bedrock-mantle の OpenAI 互換エンドポイントを使用することで、最小限のコード変更で既存アプリケーションを Amazon Bedrock に移行でき、Project Mantle による高パフォーマンスな分散推論の恩恵を受けられます。
💫⚡
マルチモデル比較・最適化
複数のオープンウェイトモデルを評価し、特定のユースケースに最適なモデルを選択したい。
7 社のプロバイダのモデルが同一リージョンで利用可能なため、統一された API を通じて簡単にモデルを切り替え、パフォーマンスやコストを比較検証できます。
