AWS Config - 新しいリソースタイプをサポート

+30

2026 年 2 月 6 日

概要

AWS Config が 30 の新しい AWS リソースタイプをサポートしました。Amazon EKS、Amazon Q、AWS IoT などの主要サービスを含むこの拡張により、AWS 環境に対するより広範なカバレッジが提供され、より効果的にリソースの検出、評価、監査、修復が可能になります。全リソースタイプの記録を有効にしている場合、自動的に追跡が開始されます。

Before - アップデート前の課題
  • 一部のリソースタイプが AWS Config の追跡対象外
  • カバレッジの欠如により、コンプライアンス監査に盲点
  • カスタムルールや手動チェックが必要
After - アップデート後の改善
  • 30 の追加リソースタイプで監視カバレッジが拡大
  • より包括的なコンプライアンス監査が可能に
  • Config ルールとアグリゲーターで新リソースを活用

主要機能

🔎
自動追跡
全リソースタイプの記録を有効にしている場合、自動的に新リソースを追跡。追加設定不要。
✅
Config ルール統合
新リソースタイプに対してカスタムルールを作成可能。既存マネージドルールも順次対応。
🌐
アグリゲーター対応
マルチアカウント、マルチリージョンでの集約が可能。組織全体でのコンプライアンス可視化。

ユースケース

EKS セキュリティ監査
EKS Nodegroup のセキュリティ設定を継続的に監査。カスタム Config ルールでリモートアクセス設定などをチェックし、セキュリティポリシーへの準拠を自動検証。
IoT デバイス管理
IoT デバイスと Topic Rule の設定を一元管理。設定変更履歴を追跡し、監査要件を満たす。
Amazon Q Business ガバナンス
組織内の Amazon Q Business アプリケーションを可視化。生成 AI アプリケーションの利用状況を把握し、ガバナンスを強化。
