概要

Amazon EC2 R8a インスタンスが AWS Asia Pacific (Tokyo) リージョンで利用可能になりました。R8a インスタンスは第 5 世代 AMD EPYC プロセッサ Turin を搭載し、最大周波数 4.5 GHz を実現するメモリ最適化インスタンスです。

R7a インスタンスと比較して最大 30% 高いパフォーマンスと最大 19% 優れたプライスパフォーマンスを提供します。メモリ帯域幅は R7a 比で 45% 向上しており、レイテンシに敏感なワークロードに最適です。

R8a インスタンスは SAP 認定を取得しており、ベアメタル 2 サイズを含む 12 サイズで提供されます。第 6 世代 AWS Nitro Cards 上に構築され、SQL/NoSQL データベース、分散キャッシュ、インメモリデータベース、リアルタイムビッグデータ分析、電子設計自動化 (EDA) などのワークロードに適しています。

Before / After

Before

  • 東京リージョンのお客様は R7a や他のメモリ最適化インスタンスタイプを使用する必要があった
  • 日本のお客様が AMD EPYC Turin プロセッサの性能をメモリ最適化ワークロードで活用できなかった
After
  • Asia Pacific (Tokyo) で R8a インスタンスが利用可能に
  • R7a からの移行で最大 30% のパフォーマンス向上と 19% のプライスパフォーマンス改善を実現
  • SAP 認定インスタンスが東京リージョンで利用可能になり、メモリ集約型ワークロードに対応

主要機能

第 5 世代 AMD EPYC プロセッサ Turin
  • 最大周波数 4.5 GHz
  • 各 vCPU は物理 CPU コア (SMT なし)
  • AMD Secure Memory Encryption (SME) による AES-256 暗号化で常時メモリ暗号化
パフォーマンス改善
  • R7a 比で最大 30% 高いパフォーマンス
  • R7a 比で 45% 高いメモリ帯域幅
  • GroovyJVM: 最大 60% 高速
  • 最大 19% 優れたプライスパフォーマンス
高性能インターフェース
  • ネットワーク帯域幅: 最大 75 Gbps (R7a 比 50% 向上)
  • EBS 帯域幅: 最大 60 Gbps
  • インスタンスあたり最大 128 EBS ボリュームをサポート
  • Instance Bandwidth Configuration (IBC) 機能でネットワークまたは EBS 帯域幅を 25% ブースト可能
SAP 認定
  • R7a 比で 38% 多い SAPS を提供
  • ミッションクリティカルな SAP ワークロードに対応

対応命令セット

R8a インスタンスは AVX-512、VNNI、bfloat16 命令セットをサポートしており、畳み込みニューラルネットワークベースのアルゴリズム、金融分析、動画エンコーディングの効率的な実行が可能です。

設定方法

前提条件

  • Asia Pacific (Tokyo) リージョンの AWS アカウントを保有していること
  • EC2 インスタンスの起動権限を持つ IAM ユーザーまたはロールがあること
  • 対象リージョンでの EC2 サービスクォータが確認済みであること

ステップ 1: AWS CLI でインスタンスを起動

Bash R8a インスタンスの起動 
aws ec2 run-instances \
  --instance-type r8a.xlarge \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --key-name my-key-pair \
  --security-group-ids sg-xxxxxxxxxxxxxxxxx \
  --subnet-id subnet-xxxxxxxxxxxxxxxxx \
  --region ap-northeast-1

ステップ 2: 利用可能なインスタンスサイズを確認

Bash R8a インスタンスタイプの一覧表示 
aws ec2 describe-instance-types \
  --filters "Name=instance-type,Values=r8a.*" \
  --region ap-northeast-1 \
  --query "InstanceTypes[].{Type:InstanceType,vCPU:VCpuInfo.DefaultVCpus,Memory:MemoryInfo.SizeInMiB}" \
  --output table

ステップ 3: 購入オプションを選択

R8a インスタンスは以下の購入オプションで利用可能です。

  • On-Demand: 長期契約なしで時間単位の料金
  • Savings Plans: 1 年または 3 年のコミットメントで割引
  • Spot インスタンス: 未使用キャパシティを大幅な割引で利用

メリット・デメリット

メリット

ビジネス面

  • R7a 比で最大 19% 優れたプライスパフォーマンスにより運用コストを削減
  • 日本国内のデータレジデンシー要件に対応しながら最新インスタンスを利用可能
  • SAP 認定で R7a 比 38% 多い SAPS を提供し、ミッションクリティカルな SAP ワークロードを東京で実行可能
  • 12 サイズ (1 vCPU から 192 vCPU) で多様なワークロードに対応

技術面

  • AMD EPYC Turin プロセッサで 4.5 GHz の最大周波数を実現
  • R7a 比 45% のメモリ帯域幅向上でデータ集約型ワークロードが高速化
  • IBC 機能でネットワークまたは EBS 帯域幅を 25% ブースト可能
  • AMD SME による AES-256 メモリ暗号化でセキュリティを強化

制約・注意点

制限事項

  • R8a インスタンスはインスタンスストレージを持たない (EBS のみ)
  • ローカル NVMe ストレージが必要な場合は他のインスタンスファミリーを検討する必要がある

考慮すべき点

  • R7a からの移行時はアプリケーションの互換性テストを実施することを推奨
  • 既存の予約インスタンスや Savings Plans の適用状況を確認し、コスト最適化を図る
  • Intel ベースのメモリ最適化インスタンスが必要な場合は R8i インスタンスも検討すべき

ユースケース

ユースケース 1: SQL/NoSQL データベース
日本国内の企業が、大規模なデータベースワークロードを東京リージョンで稼働させたいシナリオ。
Bash 大規模データベース向けインスタンス起動 
aws ec2 run-instances \
  --instance-type r8a.24xlarge \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --region ap-northeast-1 \
  --placement AvailabilityZone=ap-northeast-1a \
  --network-interfaces "DeviceIndex=0,SubnetId=subnet-xxx,Groups=sg-xxx"
R7a 比で最大 30% のパフォーマンス向上と 45% のメモリ帯域幅向上により、データベースのクエリ応答時間が大幅に改善
ユースケース 2: インメモリキャッシュとデータベース
分散キャッシングフリートやインメモリデータベースを東京リージョンで運用するシナリオ。
Bash Spot インスタンスでコスト最適化 
aws ec2 run-instances \
  --instance-type r8a.8xlarge \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --region ap-northeast-1 \
  --instance-market-options MarketType=spot
45% 向上したメモリ帯域幅により、インメモリ処理のパフォーマンスが大幅に改善。Spot インスタンスを活用することでコストを最適化
ユースケース 3: SAP ワークロード
日本の製造業企業が SAP ワークロードを R8a インスタンス上で稼働するシナリオ。
Bash SAP HANA 向け大規模インスタンス起動 
aws ec2 run-instances \
  --instance-type r8a.48xlarge \
  --image-id ami-sap-hana-xxxxxxxxx \
  --region ap-northeast-1 \
  --block-device-mappings "DeviceName=/dev/sda1,Ebs={VolumeSize=1000,VolumeType=gp3,Iops=16000}"
SAP 認定の 192 vCPU、1536 GiB メモリの大規模インスタンスで、R7a 比 38% 多い SAPS を提供。メモリ集約型 SAP アプリケーションの処理速度が向上

料金

R8a インスタンスの料金はリージョンとインスタンスサイズにより異なります。On-Demand、Savings Plans、Spot インスタンスで購入可能です。最大 19% のプライスパフォーマンス改善により、同等の処理をより低コストで実行できます。

購入オプション 特徴
On-Demand 長期契約なし、時間単位の料金
Savings Plans 1 年または 3 年のコミットメントで割引
Spot インスタンス 未使用キャパシティを大幅な割引で利用

詳細な料金については、Amazon EC2 料金ページを参照してください。
