📋 概要

Amazon SageMaker Unified Studio の Visual ETL に、データプレビュー v2.0 が導入されました。ブラウザ内クエリエンジンを使用して約 1 秒でデータプレビュー結果を返すことができ、従来の Spark ベースのプレビューと比較して大幅に高速化されています。

ソースデータがブラウザにキャッシュされ、サーバーサイドの Spark セッションが不要になるため、追加のコンピューティングコストも発生しません。CSV、Parquet、JSON (Amazon S3 上) に加え、Amazon Redshift、S3 Tables、AWS Glue Data Catalog、さらに Snowflake、MySQL、PostgreSQL、SQL Server、Oracle、BigQuery、DynamoDB、DocumentDB もサポートしています。

❌ Before - 従来のプレビュー
  • Apache Spark セッションの起動が毎回必要で、結果取得に時間がかかっていた
  • Spark セッションの起動と維持に追加のコンピューティングコストが発生
  • データ確認と検証サイクルが遅く、開発効率が低下
✅ After - v2.0
  • ブラウザ内クエリエンジンにより約 1 秒でプレビュー結果を取得
  • Spark セッション不要で追加コンピューティングコストなし
  • ブラウザキャッシュで繰り返しプレビューがさらに高速化
  • v2.0 と従来の Spark ベースプレビューをトグルで切り替え可能

🚀 主要機能

⚡
ブラウザ内クエリエンジン
サーバーサイドの Spark セッションを使用せず、ブラウザ内でデータプレビューを実行。約 1 秒でプレビュー結果を取得でき、追加コンピューティングコストも不要。
🗄️
ブラウザキャッシュ
ソースデータがブラウザにキャッシュされ、繰り返しのプレビューではキャッシュからデータを即時取得。ETL パイプライン開発中のデータ確認サイクルを大幅に短縮。
🔌
幅広いデータソースサポート
Amazon S3 上の CSV / Parquet / JSON、Amazon Redshift、S3 Tables、AWS Glue Data Catalog に加え、Snowflake、MySQL、PostgreSQL、Oracle、BigQuery など主要サードパーティソースに対応。
🔄
プレビューモード切替
v2.0 (ブラウザ内クエリエンジン) と従来の Spark ベースプレビューをトグルで切り替え可能。必要に応じて従来のプレビュー方式も利用できます。

🛠️ 設定方法

前提条件

  • ✔️ Amazon SageMaker Unified Studio ドメインが作成されていること
  • ✔️ Visual ETL プロジェクトが存在すること
  • ✔️ 対象データソースへのアクセス権限が設定されていること

手順

1
Visual ETL ジョブを開く
SageMaker Unified Studio にログインし、Visual ETL プロジェクトを開きます。既存のジョブまたは新規ジョブを選択します。
2
データソースノードを追加
Visual ETL キャンバスで、対象のデータソースノードを追加し、接続設定を行います。
3
データプレビュー v2.0 を使用
データソースノードを選択し、データプレビューを実行します。デフォルトで v2.0 (ブラウザ内クエリエンジン) が使用されます。必要に応じて、トグルスイッチで従来の Spark ベースプレビューに切り替えることも可能です。

✨ メリット

💼 ビジネス面

  • ✔️ 開発効率の大幅な向上: プレビュー待ち時間が短縮され、ETL パイプラインの開発サイクルが高速化
  • ✔️ コスト削減: Spark セッション不要で追加コンピューティングコストなし
  • ✔️ 迅速なデータ検証: 約 1 秒でプレビュー結果が得られ、品質確認を素早く繰り返し可能

💻 技術面

  • ✔️ ブラウザ内処理: クライアントサイドで完結し、サーバーリソースへの依存が減少
  • ✔️ キャッシュ活用: 同一データへの繰り返しアクセスが高速化
  • ✔️ 幅広い互換性: AWS ネイティブサービスに加え、主要サードパーティデータソースをサポート

⚠️ 制約事項・考慮点

⚠️ 制限事項

  • ⚠️ ブラウザ内クエリエンジンで処理可能なデータ量にはクライアント側メモリに依存した制限がある可能性
  • ⚠️ 大規模データセットでは従来の Spark ベースプレビューの方が適切な場合あり
  • ⚠️ ブラウザキャッシュのサイズ上限により、非常に大きなデータセットではキャッシュが効かない可能性

💡 考慮すべき点

  • 💡 ブラウザキャッシュにデータ保存されるため、機密性の高いデータはセキュリティポリシーの確認を推奨
  • 💡 従来の Spark ベースプレビューが必要なケースに備え、両方のモードの使い分けを理解しておくことが重要
