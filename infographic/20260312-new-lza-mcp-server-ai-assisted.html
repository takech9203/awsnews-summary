📋 概要

Landing Zone Accelerator on AWS (LZA) の Model Context Protocol (MCP) Server がオープンソースとして公開された。AI アシスタントとの自然言語対話を通じて、LZA デプロイメントの設定管理、パイプライン監視、トラブルシューティングを効率化する。

🔄 Before / After

Before - 従来の課題
  • 手動で YAML/JSON ファイルを編集し、時間と労力が必要
  • LZA スキーマの理解にドキュメントの横断的な調査が必要
  • パイプライン失敗時の原因特定プロセスが煩雑
  • 複数の LZA バージョン間のスキーマ差異把握が困難
After - MCP Server 導入後
  • 自然言語対話で LZA 設定タスクを効率化
  • 20 の専用ツールで検索・管理・監視を統合
  • デプロイ失敗時に自動診断とガイダンスを提供
  • Universal Configuration でコンプライアンス準拠の迅速デプロイ

⚙️ 主要機能

🔍 スキーマ検索・ディスカバリ

複数の LZA バージョンのスキーマを検索可能。プロパティ名、パターン、複雑性による検索をサポートし、ビルド時プリプロセスによりサブ秒での検索を実現。

⚙️ 設定管理

S3 からの LZA 設定ファイルの取得とアップロード。最小構成テンプレートの生成、Universal Configuration テンプレートとの統合マージに対応。

🔄 パイプライン管理

パイプライン実行の開始とステータス監視。失敗時の詳細なエラーログ取得と診断、CodeBuild ログの分析とトラブルシューティングガイダンスを提供。

📋 Universal Configuration 統合

エンタープライズ向けコンプライアンステンプレートの統合。Hub-and-Spoke、Shared VPC などのネットワークパターンをサポート。

Step 1: コンテナイメージのビルド

Bash ビルドコマンド (約 30-60 秒) 
cd src/lza-mcp-server
make build

Step 2: AWS 認証情報の設定

Bash IAM Identity Center でのログイン 
# IAM Identity Center を使用する場合
aws sso login --profile your-sso-profile

Step 3: MCP クライアントの設定

JSON IDE の MCP 設定ファイル 
{
  "mcpServers": {
    "awslabs.lza-mcp-server": {
      "command": "<REPO_PATH>/scripts/extract-aws-credentials.sh",
      "args": [
        "docker", "run",
        "--security-opt=no-new-privileges:true",
        "--cap-drop=ALL", "--read-only",
        "--tmpfs", "/tmp:rw,noexec,nosuid,size=200m",
        "--rm", "-i",
        "-v", "<CONFIG_PATH>:/app/lza-config:rw",
        "-e", "AWS_ACCESS_KEY_ID",
        "-e", "AWS_SECRET_ACCESS_KEY",
        "-e", "AWS_SESSION_TOKEN",
        "-e", "AWS_REGION",
        "lza-mcp-server:local"
      ],
      "env": {
        "AWS_PROFILE": "<YOUR_AWS_PROFILE>",
        "AWS_REGION": "<YOUR_AWS_REGION>"
      }
    }
  }
}

Step 4: 接続の検証

Prompt IDE を再起動後、以下を実行

⚠️ 制限事項・考慮すべき点

制限事項

  • CodeCommit ベースの設定リポジトリは現在サポートされていない (S3 ベースのみ対応)
  • LZA v1.12.0 以降のバージョンのみサポート
  • コンテナランタイム (Docker または Finch) が必須

考慮すべき点

  • AWS API レスポンスが AI モデルプロバイダーに共有されるため、組織のセキュリティ・プライバシーポリシーとの整合性を確認する必要あり
  • IAM Identity Center のセッションは通常 8-12 時間で期限切れ。定期的な再認証が必要
  • ビルド時に含める LZA バージョン数が増えると、イメージサイズとビルド時間が比例して増加
