概要

Amazon EC2 R8a インスタンスが AWS Europe (Ireland) リージョンで利用可能になりました。R8a インスタンスは第 5 世代 AMD EPYC プロセッサ (Turin) を搭載し、最大周波数 4.5 GHz を実現するメモリ最適化インスタンスです。

R7a インスタンスと比較して最大 30% 高いパフォーマンスと最大 19% 優れたプライスパフォーマンスを提供します。メモリ帯域幅は R7a 比で 45% 向上しており、ベアメタル 2 サイズを含む 12 サイズで提供されます。第 6 世代 AWS Nitro Cards 上に構築され、SAP 認定を取得しています。

Before - アップデート前の課題
  • Europe (Ireland) リージョンでは R8a インスタンスが利用できなかった
  • アイルランドリージョンのお客様は R7a や他のメモリ最適化インスタンスを使用する必要があった
  • 欧州のお客様が AMD EPYC Turin プロセッサの性能をメモリ最適化ワークロードで活用できるリージョンが限られていた
After - アップデート後の改善
  • Europe (Ireland) で R8a インスタンスが利用可能に
  • R7a からの移行で最大 30% のパフォーマンス向上と 19% のプライスパフォーマンス改善を実現
  • SAP 認定インスタンスがアイルランドリージョンで利用可能になり、欧州のメモリ集約型ワークロードに対応

R8a インスタンスは第 6 世代 Nitro Cards 上に構築され、AMD EPYC Turin プロセッサ、高速ネットワーク、EBS ストレージ、常時メモリ暗号化を統合しています。

主要機能

⚡
第 5 世代 AMD EPYC プロセッサ
コード名 Turin の最新プロセッサを搭載。最大周波数 4.5 GHz、各 vCPU は物理 CPU コア (SMT なし) で安定した性能を提供。
📈
メモリ帯域幅の大幅向上
R7a 比で 45% のメモリ帯域幅向上を実現。レイテンシに敏感なワークロードや大規模データセットの処理に最適。
📡
高性能インターフェース
ネットワーク帯域幅: 最大 75 Gbps、EBS 帯域幅: 最大 60 Gbps。IBC 機能でネットワークまたは EBS 帯域幅を 25% ブースト可能。
🔒
常時メモリ暗号化とセキュリティ
AMD Secure Memory Encryption (SME) による AES-256 暗号化で常時メモリ暗号化を実現。第 6 世代 AWS Nitro Cards 上に構築。

設定方法

前提条件
  • Europe (Ireland) リージョンの AWS アカウントを保有していること
  • EC2 インスタンスの起動権限を持つ IAM ユーザーまたはロールがあること
  • 対象リージョンでの EC2 サービスクォータが確認済みであること
ステップ 1: インスタンスの起動
Bash R8a インスタンスの起動 
aws ec2 run-instances \
  --instance-type r8a.xlarge \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --key-name my-key-pair \
  --security-group-ids sg-xxxxxxxxxxxxxxxxx \
  --subnet-id subnet-xxxxxxxxxxxxxxxxx \
  --region eu-west-1

Europe (Ireland) リージョンで r8a.xlarge インスタンスを起動します。

ステップ 2: 利用可能なインスタンスサイズを確認
Bash インスタンスタイプの確認 
aws ec2 describe-instance-types \
  --filters "Name=instance-type,Values=r8a.*" \
  --region eu-west-1 \
  --query "InstanceTypes[].{Type:InstanceType,vCPU:VCpuInfo.DefaultVCpus,Memory:MemoryInfo.SizeInMiB}" \
  --output table

Europe (Ireland) リージョンで利用可能な R8a インスタンスタイプとスペックを表示します。

ユースケース

🗃️ ユースケース 1: SQL/NoSQL データベース
シナリオ: 欧州の企業が、大規模なデータベースワークロードをアイルランドリージョンで稼働させたい
Bash 実装例 
aws ec2 run-instances \
  --instance-type r8a.24xlarge \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --region eu-west-1 \
  --placement AvailabilityZone=eu-west-1a \
  --network-interfaces "DeviceIndex=0,SubnetId=subnet-xxx,Groups=sg-xxx"
✔ R7a 比で最大 30% のパフォーマンス向上と 45% のメモリ帯域幅向上により、データベースのクエリ応答時間が大幅に改善
⚡ ユースケース 2: インメモリキャッシュとデータベース
シナリオ: 分散キャッシングフリートやインメモリデータベースを欧州リージョンで運用
Bash 実装例 
aws ec2 run-instances \
  --instance-type r8a.8xlarge \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --region eu-west-1 \
  --instance-market-options MarketType=spot
✔ 45% 向上したメモリ帯域幅により、インメモリ処理のパフォーマンスが大幅に改善。Spot インスタンスを活用でコスト最適化
🏭 ユースケース 3: SAP ワークロード
シナリオ: 欧州の製造業企業が SAP ワークロードを R8a インスタンス上で稼働
Bash 実装例 
aws ec2 run-instances \
  --instance-type r8a.48xlarge \
  --image-id ami-sap-hana-xxxxxxxxx \
  --region eu-west-1 \
  --block-device-mappings "DeviceName=/dev/sda1,Ebs={VolumeSize=1000,VolumeType=gp3,Iops=16000}"
✔ SAP 認定の 192 vCPU、1536 GiB メモリの大規模インスタンスで、R7a 比 38% 多い SAPS を提供。メモリ集約型 SAP アプリケーションの処理速度が向上

メリット

💰 ビジネス面
  • コスト最適化: R7a 比で最大 19% 優れたプライスパフォーマンスにより運用コストを削減
  • 欧州リージョン対応: GDPR 等の欧州データレジデンシー要件に対応しながら最新インスタンスを利用可能
  • SAP 認定: R7a 比で 38% 多い SAPS を提供し、ミッションクリティカルな SAP ワークロードをアイルランドリージョンで実行可能
  • 柔軟なスケーリング: 12 サイズで多様なワークロードに対応
⚡ 技術面
  • 高い CPU 性能: AMD EPYC Turin プロセッサで 4.5 GHz の最大周波数を実現
  • メモリ帯域幅の向上: R7a 比 45% のメモリ帯域幅向上により、データ集約型ワークロードが高速化
  • IBC 機能: Instance Bandwidth Configuration でネットワークまたは EBS 帯域幅を 25% ブースト可能
  • 常時暗号化: AMD SME による AES-256 メモリ暗号化でセキュリティを強化

制約事項・考慮点

⚠ 制限事項
  • R8a インスタンスはインスタンスストレージを持たない (EBS のみ)
  • ローカル NVMe ストレージが必要な場合は他のインスタンスファミリーを検討する必要がある
ℹ 考慮すべき点
  • R7a からの移行時は、アプリケーションの互換性テストを実施することを推奨
  • 既存の予約インスタンスや Savings Plans の適用状況を確認し、コスト最適化を図る
  • Intel ベースのメモリ最適化インスタンスが必要な場合は、R8i インスタンスも検討すべき
