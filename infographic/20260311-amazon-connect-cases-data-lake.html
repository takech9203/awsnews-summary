Amazon Connect Cases

📊 分析データレイクでのケースデータ提供

Cases Data Now Available in Analytics Data Lake
2026 年 3 月 11 日

📋 概要

Amazon Connect Cases のデータが分析データレイクで利用可能になりました。ケースデータを他の Amazon Connect 分析データと組み合わせて、Amazon Athena や Amazon QuickSight を使用したカスタムレポートやインサイトの生成が容易になります。

ケースタイプ別のボリューム分析、エージェントシフトをまたいだケース対応状況、ケース全体のコンタクト感情分析などのトレンド分析が可能となり、複雑なデータパイプラインの構築・維持が不要になる点が大きなメリットです。

❌ Before
•ケースデータを他の分析データと統合するには独自のデータパイプラインが必要
•トレンド分析やクロスデータ分析のために ETL プロセスの開発・運用が必要
•ケース・コンタクト・エージェントデータを横断的に分析する手段が限定的
✅ After
•ケースデータが分析データレイクに自動的に含まれ、カスタムパイプライン不要
•Amazon Athena で SQL クエリを使用してケースデータを直接分析可能
•Amazon QuickSight と連携してビジュアルダッシュボードを作成可能

⚙ 主要機能

🗄 ケースデータのデータレイク統合
  • Amazon Connect Cases のケースデータが分析データレイクに自動的に格納される
  • 既存のコンタクトデータやエージェントパフォーマンスデータと同じデータレイクで管理
  • 追加のデータパイプライン構築なしでデータが利用可能
🔍 クロスデータ分析
  • ケースデータとコンタクトデータを結合したクロス分析が可能
  • ケースタイプ別ボリューム、エージェントシフト間のケース対応、コンタクト感情分析を横断的に分析
  • SQL ベースのクエリで柔軟にデータを抽出・集計
📊 BI ツールとの統合
  • Amazon Athena を使用した SQL によるアドホッククエリ
  • Amazon QuickSight によるダッシュボードやレポートの作成
  • 既存の BI ワークフローにケースデータを組み込み可能

🚀 設定方法

前提条件

→Amazon Connect インスタンスが設定済みであること
→Amazon Connect Cases が有効化されていること
→分析データレイクが有効化されていること

手順

1
分析データレイクの有効化確認
Amazon Connect コンソールにアクセスし、分析データレイクが有効になっていることを確認します。未設定の場合は、Amazon Connect の分析設定からデータレイクを有効化してください。
2
Amazon Athena でのクエリ実行
分析データレイク内のケーステーブルに対して Athena で直接 SQL クエリを実行できます。
SQL ケースタイプ別ボリューム確認クエリ例 
-- ケースタイプ別のボリュームを確認するクエリ例
SELECT case_type, COUNT(*) as case_count
FROM connect_analytics_data_lake.cases
GROUP BY case_type
ORDER BY case_count DESC;
3
QuickSight ダッシュボードの作成
Amazon QuickSight で Athena をデータソースとして接続し、ケースデータを含むダッシュボードを作成します。既存のコンタクト分析ダッシュボードにケースデータを追加することも可能です。

✅ メリット・制約事項

ビジネス面のメリット

✓運用コストの削減: カスタムデータパイプラインの構築・運用が不要になり、インフラ管理コストを削減
✓意思決定の迅速化: ケースデータとコンタクトデータの統合分析により、運用改善の意思決定を迅速化
✓顧客体験の向上: ケースの傾向分析を通じて、顧客対応のボトルネックを特定し改善

技術面のメリット

✓データ統合の簡素化: データレイクにケースデータが自動統合、ETL パイプラインの開発が不要
✓標準 SQL によるアクセス: Amazon Athena を通じて標準 SQL でアクセスでき、学習コストが低い
✓スケーラブルな分析基盤: サーバーレスアーキテクチャにより、データ量の増加に応じて自動スケール

⚠ 制限事項・考慮点

  • 分析データレイクの有効化が前提条件となるため、未設定の場合は追加の初期設定が必要
  • データレイクへのデータ反映にはリアルタイムではなく一定のラグが発生する可能性がある
  • 利用可能リージョンが限定されている
  • Athena のクエリ実行にはスキャンしたデータ量に応じたコストが発生
  • QuickSight ダッシュボードの利用には別途 QuickSight のサブスクリプションが必要

💻 クエリサンプル

SQL ケースタイプ別の週次トレンド 
SELECT case_type,
       DATE_TRUNC('week', created_at) as week,
       COUNT(*) as case_count
FROM connect_analytics_data_lake.cases
GROUP BY case_type, DATE_TRUNC('week', created_at)
ORDER BY week DESC, case_count DESC;
SQL コンタクト感情とケース解決の相関 
SELECT c.case_type,
       AVG(ct.sentiment_score) as avg_sentiment,
       AVG(c.resolution_time_hours) as avg_resolution_hours
FROM connect_analytics_data_lake.cases c
JOIN connect_analytics_data_lake.contacts ct ON c.case_id = ct.case_id
GROUP BY c.case_type;
SQL エージェントシフト別のケース対応 
SELECT agent_shift,
       COUNT(DISTINCT case_id) as handled_cases,
       COUNT(CASE WHEN status = 'RESOLVED' THEN 1 END) as resolved_cases
FROM connect_analytics_data_lake.cases
GROUP BY agent_shift
ORDER BY agent_shift;

📝 まとめ

Amazon Connect Cases のデータが分析データレイクで利用可能になったことで、コンタクトセンターの運用データをより包括的に分析できるようになりました。カスタムデータパイプラインの構築が不要になるため、分析環境の構築コストと運用負荷が大幅に軽減されます。

Amazon Connect を利用しているコンタクトセンターでは、分析データレイクを有効化し、ケースデータを活用したトレンド分析やダッシュボード構築を検討することを推奨します。
