📋 概要

Amazon EC2 C8gd および M8gd インスタンスが追加リージョンで利用可能になりました。C8gd は南米 (サンパウロ)、M8gd はヨーロッパ (アイルランド)に拡大されています。AWS Graviton4 プロセッサーを搭載し、最大 11.4 TB のローカル NVMe SSD ストレージを備えた I/O 集約型ワークロード向けインスタンスです。

❌ Before
•サンパウロで C8gd が利用不可、Graviton4 + ローカル NVMe SSD を活用できず
•アイルランドで M8gd が利用不可、汎用 + ローカルストレージの組み合わせ不可
•前世代 Graviton3 ベースインスタンスに限定、最新パフォーマンス向上の恩恵なし
✅ After
•C8gd がサンパウロで利用可能、ローカル NVMe SSD 付きコンピューティング最適化
•M8gd がアイルランドで利用可能、汎用ワークロードにローカルストレージ活用
•帯域幅重み付け設定で NW と EBS の帯域幅を 25% 調整可能に

⚙️ 設定方法

前提条件

  • ✔ 対象リージョンの AWS アカウント (sa-east-1 または eu-west-1)
  • ✔ Arm (aarch64/arm64) アーキテクチャ対応の AMI
  • ✔ 適切な IAM 権限 (EC2 インスタンスの起動権限)

Step 1: AMI の選択

Bash Graviton 対応 AMI を検索 
# サンパウロリージョンで Graviton 対応の Amazon Linux 2023 AMI を検索
aws ec2 describe-images \
  --region sa-east-1 \
  --filters "Name=name,Values=al2023-ami-*-arm64-*" \
            "Name=state,Values=available" \
  --query 'Images | sort_by(@, &CreationDate) | [-1].ImageId' \
  --output text

Step 2: C8gd インスタンスの起動

Bash サンパウロリージョンで起動 
# C8gd インスタンスを起動
aws ec2 run-instances \
  --region sa-east-1 \
  --instance-type c8gd.xlarge \
  --image-id ami-xxxxxxxxxxxxxxxxx \
  --key-name my-key-pair \
  --security-group-ids sg-xxxxxxxxxxxxxxxxx \
  --subnet-id subnet-xxxxxxxxxxxxxxxxx

Step 3: ローカル NVMe SSD のマウント

Bash NVMe SSD のフォーマットとマウント 
# ローカル NVMe SSD デバイスを確認
lsblk

# ファイルシステムを作成してマウント
sudo mkfs -t xfs /dev/nvme1n1
sudo mkdir /mnt/local-ssd
sudo mount /dev/nvme1n1 /mnt/local-ssd

⚠️ ローカル NVMe SSD はエフェメラルストレージです。インスタンスの停止や終了時にデータは失われます。永続的なデータは Amazon EBS に保存してください。

Step 4: 帯域幅重み付けの設定 (オプション)

Bash 帯域幅重み付けを設定 
# インスタンスの帯域幅重み付けを設定
aws ec2 modify-instance-attribute \
  --instance-id i-xxxxxxxxxxxxxxxxx \
  --instance-type c8gd.xlarge \
  --bandwidth-weighting "default"

✅ メリット

💼 ビジネス面

  • ✔ リージョン選択肢の拡大: 南米やヨーロッパでローカル SSD 付き Graviton4 インスタンスを利用可能
  • ✔ コスト効率の向上: Graviton4 は x86 比で優れた価格パフォーマンス、ローカル NVMe SSD で EBS コスト削減
  • ✔ 帯域幅の柔軟な配分: ワークロード特性に応じたリソース配分でインフラコスト最適化

🔧 技術面

  • ✔ 低レイテンシーストレージ: ローカル NVMe SSD でネットワーク経由比大幅に低いレイテンシー
  • ✔ 大幅なパフォーマンス向上: DB で最大 40% 高速化、分析で最大 20% 高速化
  • ✔ EFA サポート: 大型インスタンスで高帯域幅・低レイテンシー通信、HPC/ML に最適

⚠️ 制限事項・注意点

  • ⚠️ ローカル NVMe SSD はエフェメラルストレージ — インスタンスの停止・終了でデータ消失
  • ⚠️ Arm (aarch64) アーキテクチャのみ対応 — x86 バイナリはそのまま実行不可
  • ⚠️ 今回の拡大は C8gd がサンパウロ、M8gd がアイルランドに限定
  • 💡 永続データには Amazon EBS / S3 との併用を推奨
  • 💡 x86 からの移行には Porting Advisor for Graviton で互換性テスト実施を推奨
  • 💡 帯域幅重み付け設定はワークロードに応じたベンチマークテストが望ましい

🎯 ユースケース

🇧🇷 南米地域での I/O 集約型データベース
南米市場向けアプリケーションで、高速ローカルストレージを必要とするデータベースワークロードをサンパウロリージョンで実行。C8gd.8xlarge を使用。
📈 Graviton4 による I/O 集約型 DB の最大 40% 高速化 + ローカル NVMe SSD の低レイテンシーアクセスで、南米ユーザーへ高パフォーマンスな DB サービスを提供
🇮🇪 ヨーロッパでのリアルタイムデータ分析
ヨーロッパのデータセンターでリアルタイムデータ分析を実行し、ローカルストレージを一時データ処理に活用。M8gd.16xlarge を使用。
📈 リアルタイム分析の最大 20% 高速なクエリ + M8gd のバランスの取れたリソースで効率的なデータ分析パイプラインを構築
⚡ HPC ワークロードでの EFA 活用
大規模科学計算やシミュレーションで、ノード間の高帯域幅・低レイテンシー通信とローカル高速ストレージが必要。C8gd.48xlarge + EFA を使用。
📈 48xlarge の EFA サポートで高速ノード間通信 + ローカル NVMe SSD のスクラッチスペースで効率的な HPC 実行

📝 まとめ

Amazon EC2 C8gd が南米 (サンパウロ)、M8gd がヨーロッパ (アイルランド) で利用可能になり、AWS Graviton4 + ローカル NVMe SSD の高性能ワークロードをこれらのリージョンで実行できるようになりました。I/O 集約型 DB で最大 40% 高速化、リアルタイム分析で最大 20% 高速化が期待できます。ローカル NVMe SSD を必要とするワークロードをこれらのリージョンで運用しているユーザーは、C8gd / M8gd への移行検討を推奨します。
