AWS Savings Plans

Database Savings Plans - OpenSearch Service / Neptune Analytics 対応

1 年間のコミットメントで最大 35% のコスト削減 - 対象サービス拡大でより柔軟なコスト最適化
2026 年 3 月 5 日

📝 概要

Database Savings Plans の対象サービスが拡大され、Amazon OpenSearch Service および Amazon Neptune Analytics のサポートが追加。1 年間のコミットメントで最大 35% のコスト削減が可能に。エンジン、インスタンスファミリー、サイズ、リージョンに関係なく自動適用される。

35%
最大コスト削減
1 年
コミットメント期間

🔄 Before / After

Before
❌DBSP の対象が限定的で OpenSearch / Neptune Analytics は割引対象外
❌個別にリザーブドインスタンス等の割引モデルを検討が必要
❌複数 DB サービス横断の統一的コスト最適化が困難
→
After
✅OpenSearch Service / Neptune Analytics に最大 35% 割引が自動適用
✅サーバーレスとプロビジョンドの両方が対象
✅既存 DBSP の恩恵が自動的に新サービスにも拡大

⚙️ 主要機能

OpenSearch Service サポート
🔍プロビジョンドインスタンスおよび OpenSearch Serverless が対象
🔍エンジンバージョン、インスタンスファミリー、サイズ不問
Neptune Analytics サポート
🔗OCU-hours 単位の使用量が DBSP 対象に追加
🔗リージョンを問わず自動割引適用
自動適用メカニズム
🤖最も割引率の高い使用量から自動適用
🤖エンジン、サイズ、デプロイオプション、リージョン不問

🔌 API 変更

4 つの API メソッドが更新され、OpenSearch および Neptune Analytics の Database Savings Plans サポートが追加。

  • DescribeSavingsPlanRates - productType に OpenSearch 追加、unit に Jobs / OCU-hours 追加
  • DescribeSavingsPlans - productTypes に OpenSearch 追加
  • DescribeSavingsPlansOfferingRates - products に OpenSearch 追加
  • DescribeSavingsPlansOfferings - productType に OpenSearch 追加

serviceCode AmazonES が追加対象

🛠️ 設定方法

前提条件: Savings Plans 購入権限、Cost Explorer 有効化、OpenSearch Service / Neptune Analytics 利用中または利用予定。

Step 1: 使用量を確認
Bash 推奨事項取得 
aws ce get-savings-plans-purchase-recommendation \
  --savings-plans-type "DATABASE_SAVINGS_PLANS" \
  --term-in-years "ONE_YEAR" \
  --payment-option "NO_UPFRONT" \
  --lookback-period-in-days "THIRTY_DAYS"
Step 2: オファリング確認・購入
Bash オファリング確認 
aws savingsplans describe-savings-plans-offerings \
  --product-type "OpenSearch" \
  --plan-types "SavingsPlans"
Step 3: 適用状況モニタリング
Bash Savings Plans 確認 
aws savingsplans describe-savings-plans \
  --filter '{"key":"start","values":["2026-03-05"]}'

💡 ユースケース

1. OpenSearch Service ログ分析基盤

大規模ログ分析基盤で OpenSearch Service を複数リージョンで運用しているお客様がコスト最適化を検討。

Bash
aws ce get-savings-plans-purchase-recommendation \
  --savings-plans-type "DATABASE_SAVINGS_PLANS" \
  --term-in-years "ONE_YEAR" \
  --payment-option "ALL_UPFRONT" \
  --lookback-period-in-days "SIXTY_DAYS"

効果: 月額 $10,000 の使用量で最大 $3,500/月削減

2. Neptune Analytics グラフ分析

不正検知やレコメンデーションエンジンで Neptune Analytics を定常利用。予測可能なワークロードのコスト最適化。

Bash
aws ce get-cost-and-usage \
  --time-period Start=2026-02-01,End=2026-03-01 \
  --granularity MONTHLY \
  --metrics "UsageQuantity" \
  --filter '{"Dimensions":{"Key":"SERVICE","Values":["Amazon Neptune"]}}'

効果: OCU-hours 使用量に最大 35% 割引適用

3. マルチ DB 環境の統合最適化

RDS、Aurora、OpenSearch Service、Neptune Analytics を組み合わせ利用。個別 RI ではなく統合的なコスト最適化。

Bash
aws ce get-savings-plans-utilization \
  --time-period Start=2026-03-01,End=2026-03-07 \
  --granularity DAILY \
  --filter '{"Dimensions":{"Key":"SAVINGS_PLAN_TYPE","Values":["DatabaseSavingsPlans"]}}'

効果: 単一プランで使用量変動を吸収し最大割引

🔗 関連サービス・機能

🔍Amazon OpenSearch Service: フルマネージドの検索・分析サービス。ログ分析、全文検索、モニタリングに使用
🔗Amazon Neptune Analytics: グラフデータに対する高速分析クエリサービス。不正検知やナレッジグラフに活用
📊AWS Cost Explorer: コスト分析と Savings Plans 推奨事項、利用率モニタリングに使用
⚡Compute Savings Plans: EC2、Lambda、Fargate に対する同様の柔軟な割引モデル

📋 まとめ

Database Savings Plans の対象サービスに Amazon OpenSearch Service と Amazon Neptune Analytics が追加。1 年間のコミットメントで最大 35% のコスト削減が可能に。OpenSearch Service や Neptune Analytics を定常的に利用しているお客様は、Cost Explorer の推奨事項を確認し、適切なコミットメント額での Savings Plans 購入を検討することを推奨。
