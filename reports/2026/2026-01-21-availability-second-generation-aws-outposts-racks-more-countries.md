# AWS Outposts racks - 第 2 世代の 20 カ国への展開

**リリース日**: 2026 年 1 月 21 日
**サービス**: AWS Outposts
**機能**: 第 2 世代 Outposts racks の地理的拡大

## 概要

第 2 世代 AWS Outposts racks が、新たに 20 カ国のデータセンターおよびオンプレミス環境に出荷および設置可能になりました。対象国は、アルゼンチン、バングラデシュ、コロンビア、ドミニカ共和国、エクアドル、インド、カザフスタン、メキシコ、モロッコ、ナイジェリア、オマーン、パナマ、カタール、セネガル、セルビア、南アフリカ、韓国、台湾、タイ、ウルグアイです。

AWS Outposts racks は、AWS のインフラストラクチャ、サービス、API、ツールを、事実上あらゆるオンプレミスのデータセンターまたはコロケーションスペースに拡張し、真の一貫したハイブリッドエクスペリエンスを提供します。第 2 世代 Outposts racks は、最新世代の x86 ベースの Amazon EC2 インスタンス（C7i、M7i、R7i インスタンスなど）をサポートし、第 1 世代の C5、M5、R5 インスタンスと比較して最大 40% 優れたパフォーマンスを提供します。

**アップデート前の課題**

- 第 2 世代 Outposts racks を利用できる国が限られていた
- 一部の地域では、データレジデンシー要件を満たしながら最新世代の EC2 インスタンスを利用できなかった
- オンプレミスで AWS サービスを実行する際に、最新のパフォーマンスとネットワーク機能を利用できない地域があった

**アップデート後の改善**

- 20 の新しい国で第 2 世代 Outposts racks が利用可能になり、グローバルな展開が容易に
- データレジデンシー要件を満たしながら、最新世代の高性能 EC2 インスタンスをオンプレミスで実行可能に
- より多くの地域で、簡素化されたネットワークスケーリングと構成、超低レイテンシーと高スループットに最適化された新しいアクセラレーテッドネットワーキング EC2 インスタンスを利用可能に

## サービスアップデートの詳細

### 主要機能

1. **最新世代 EC2 インスタンスのサポート**
   - C7i、M7i、R7i インスタンスなど、最新世代の x86 ベースの EC2 インスタンスをサポート
   - 第 1 世代の C5、M5、R5 インスタンスと比較して最大 40% のパフォーマンス向上
   - より高いコンピューティング能力とメモリ帯域幅

2. **簡素化されたネットワークスケーリングと構成**
   - ネットワークの拡張と構成管理が容易に
   - 運用の複雑さを軽減
   - より柔軟なネットワークアーキテクチャの実装が可能

3. **アクセラレーテッドネットワーキング EC2 インスタンス**
   - 超低レイテンシーと高スループットに最適化された新しいカテゴリのインスタンス
   - ネットワーク集約型ワークロードのパフォーマンスを向上
   - リアルタイムアプリケーションやデータ処理に最適

4. **グローバルな展開**
   - 20 の新しい国で利用可能
   - 各国でデータレジデンシーを維持しながら AWS サービスをローカルで実行
   - サポートされている AWS リージョンに接続して管理と運用を実行

## メリット

### ビジネス面

- **データレジデンシー要件への対応**: 自国内でデータを保持しながら AWS サービスを利用可能
- **グローバル展開の加速**: より多くの国で一貫した AWS 体験を提供し、グローバル展開を促進
- **規制要件への準拠**: 各国の規制やコンプライアンス要件を満たしながらクラウドサービスを利用可能

### 技術面

- **ハイブリッドクラウドアーキテクチャ**: オンプレミスと AWS リージョンをシームレスに統合
- **低レイテンシーアクセス**: オンプレミスシステムへの低レイテンシーアクセスが必要なワークロードに最適
- **パフォーマンス向上**: 最新世代のインスタンスにより、ワークロードのパフォーマンスが大幅に向上

## ユースケース

### ユースケース 1: データレジデンシー要件への対応

**シナリオ**: 金融機関が規制により顧客データを自国内に保持する必要があるが、AWS のクラウドサービスを利用したい。

**効果**: 第 2 世代 Outposts racks をオンプレミスに設置することで、データを自国内に保持しながら、AWS のマネージドサービスを利用できる。最新世代の EC2 インスタンスにより、高いパフォーマンスを維持できる。

### ユースケース 2: 低レイテンシーが必要なワークロード

**シナリオ**: 製造業が工場の IoT デバイスからリアルタイムでデータを収集し、迅速に分析および応答する必要がある。

**効果**: Outposts racks を工場に設置することで、IoT デバイスへの低レイテンシーアクセスを実現し、リアルタイムデータ処理が可能になる。アクセラレーテッドネットワーキングインスタンスにより、高スループットのデータ処理を実行できる。

### ユースケース 3: レガシーシステムとの統合

**シナリオ**: 企業がオンプレミスのレガシーシステムに依存しているが、クラウドへの移行を段階的に進めたい。

**効果**: Outposts racks をオンプレミスに設置することで、レガシーシステムとの低レイテンシー接続を維持しながら、AWS のマネージドサービスを利用できる。段階的な移行が可能になる。

## デメリット・制約事項

### 制限事項

- Outposts racks の設置には物理的なスペース、電源、冷却設備が必要
- 初期投資コストが発生する
- サポートされている AWS リージョンへの接続が必要

### 考慮すべき点

- データセンターの物理的要件（スペース、電源容量、冷却能力など）を事前に確認する必要がある
- ネットワーク接続の要件（AWS リージョンへの安定した接続）を満たす必要がある
- 保守とアップデートは AWS が管理するが、物理的なセキュリティはお客様の責任

## 料金

AWS Outposts racks の料金は、選択したインスタンスタイプ、ストレージ容量、ネットワーク構成によって異なります。3 年間の契約が標準で、月額料金が発生します。

詳細な料金については、[AWS Outposts 料金ページ](https://aws.amazon.com/outposts/pricing/) を参照してください。
