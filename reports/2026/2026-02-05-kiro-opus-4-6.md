# Kiro - Claude Opus 4.6 モデルサポート

**リリース日**: 2026 年 2 月 5 日
**サービス**: Kiro
**機能**: Claude Opus 4.6 サポート

📊 [このアップデートのインフォグラフィックを見る](https://takech9203.github.io/aws-news-summary/20260205-kiro-opus-4-6.html)

## 概要

Kiro IDE および CLI が Claude Opus 4.6 をサポートしました。Claude Opus 4.6 は Anthropic が提供する最新かつ最も強力なモデルであり、コーディング領域において世界最高水準の性能を誇ります。大規模なコードベースや長期間にわたるプロジェクトに特に優れた能力を発揮します。

Opus 4.6 は Opus 4.5 の全機能を維持しつつ、プロダクションコードや高度なエージェントに対するコーディング能力をさらに拡張しています。Kiro のスペック駆動開発ワークフローにおいて最も適したモデルであり、大規模な既存プロジェクトに対して詳細かつ正確なスペックを作成し、最小限のユーザー入力で的確な変更を行います。

**アップデート前の課題**

- Kiro で利用可能なモデルは、大規模コードベースでの複雑なタスクにおいて追加の監視が必要だった
- スペック駆動開発において、大規模プロジェクトへの精密な変更に改善の余地があった
- シニアエンジニアが複数日かかるプロジェクトを効率的に委任するには限界があった

**アップデート後の改善**

- Claude Opus 4.6 により大規模コードベースと長期プロジェクトで優れた性能を発揮
- スペック駆動開発において、大規模既存プロジェクトに対する精密な変更が可能に
- シニアエンジニアが複数日かかるプロジェクトを数時間で完了できるようになった

## サービスアップデートの詳細

### 主要機能

1. **Claude Opus 4.6 モデル統合**
   - Kiro IDE と CLI の両方で Claude Opus 4.6 を選択可能

   - 実験的サポートとして Kiro Pro、Pro+、Power のすべてのプランで利用可能

2. **大規模コードベース対応**
   - 大規模なコードベースでの高精度な理解と変更
   - 長期間にわたるプロジェクトでの一貫した品質
   - 少ない監視で複雑なタスクの委任が可能

3. **スペック駆動開発の強化**
   - 大規模既存プロジェクトに対する詳細かつ正確なスペック作成
   - 最小限のユーザー入力で的確な変更を実行
   - 既存コードへの外科的精度での更新

## 設定方法

### 前提条件

1. Kiro IDE または CLI がインストール済みであること
2. Kiro Pro、Pro+、または Power プランに加入していること
3. Google、GitHub、AWS Builder ID、または AWS IAM Identity Center のアカウント

### 手順

#### ステップ 1: Kiro の更新

```bash
# Kiro IDE の場合: アプリケーションを再起動して最新バージョンに更新
# Kiro CLI の場合: 最新バージョンをダウンロード
```

Kiro IDE を使用している場合はアプリケーションを再起動します。CLI を使用している場合は最新バージョンをダウンロードしてインストールします。

#### ステップ 2: モデルの選択

Kiro IDE または CLI のモデル選択メニューから Claude Opus 4.6 を選択します。

#### ステップ 3: 利用開始

選択後、Claude Opus 4.6 を使用してコーディングタスクやスペック駆動開発を開始できます。クレジット乗数は 2.2x が適用されます。

## メリット

### ビジネス面

- **開発期間の大幅短縮**: シニアエンジニアが複数日かかるプロジェクトを数時間で完了
- **監視コストの削減**: 少ない監視で複雑なタスクを委任可能
- **生産性の向上**: 世界最高水準のコーディングモデルによる開発効率化

### 技術面

- **大規模コードベース対応**: 大規模なコードベースでの高精度な理解と変更
- **スペック駆動開発の精度向上**: 大規模プロジェクトに対する詳細かつ正確なスペック作成
- **高度なエージェント能力**: プロダクションコードや複雑なエージェントワークフローに対応

## デメリット・制約事項

### 制限事項

- 実験的サポートのため、安定性に関して改善の余地がある
- クレジット乗数が 2.2x であり、通常モデルより多くのクレジットを消費
- モデル推論は AWS US-East-1 リージョンのみで実行

### 考慮すべき点

- クレジット消費量が増加するため、使用量の管理が必要
- 実験的サポートのため、今後の仕様変更の可能性がある
- 全てのタスクに Opus 4.6 が必要ではなく、タスクの複雑さに応じたモデル選択が推奨

## ユースケース

### ユースケース 1: 大規模リファクタリングプロジェクト

**シナリオ**: 数万行のレガシーコードベースをモダンなアーキテクチャに移行するプロジェクト

**効果**: Opus 4.6 の大規模コードベース理解能力により、広範囲にわたるリファクタリングを高い精度で実行し、通常数日かかる作業を数時間に短縮

### ユースケース 2: スペック駆動の新機能開発

**シナリオ**: 大規模既存プロジェクトに対する新機能のスペック作成と実装

**効果**: Opus 4.6 が既存コードベースを深く理解した上で詳細なスペックを作成し、既存コードとの整合性を保ちながら外科的精度で変更を適用

### ユースケース 3: 高度なエージェントワークフロー

**シナリオ**: 複数のサブエージェントを活用した複雑な開発ワークフロー

**効果**: Opus 4.6 の高度なエージェント能力により、複雑なマルチステップタスクをより少ない監視で確実に実行
