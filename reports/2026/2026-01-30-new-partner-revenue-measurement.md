# AWS Partner Revenue Measurement

**リリース日**: 2026 年 1 月 30 日
**サービス**: AWS Partner Revenue Measurement
**機能**: パートナーソリューション影響度の可視化

## 概要

AWS Partner Revenue Measurement が正式リリースされ、AWS パートナーがパートナーマネージドアカウントおよび顧客マネージドアカウント全体を通じて、パートナーソリューションが AWS サービス利用にどの程度影響しているかを可視化できるようになりました。

この機能により、パートナーは AWS 収益への影響と製品消費パターンをより詳細に理解し、ビジネス成長の機会をより正確に把握することができます。

**アップデート前の課題**

- AWS Marketplace で提供するソリューションが実際にどの程度 AWS サービス利用につながっているかが不明確だった
- パートナーの AWS 収益への直接的な影響を定量化する仕組みがなかった
- マーケットプレイス販売とそれに伴う AWS サービス利用の相関関係の追跡が困難だった

**アップデート後の改善**

- AWS Marketplace の商品コードを使用したリソースタグにより、ソリューション影響度を正確に追跡可能に
- パートナーがソリューションの AWS 収益貢献度を定量化できるようになった
- ビジネス意思決定と投資効果測定がデータ駆動型になった

## サービスアップデートの詳細

### 主要機能

1. **リソースタギングによる追跡**
   - AWS Marketplace 商品コードを使用したタグを AWS リソースに適用
   - タグキー: `aws-apn-id`
   - タグ値: `pc:<AWS Marketplace product-code>`

2. **収益インパクト測定**
   - ソリューション経由で発生した AWS サービス利用を追跡
   - パートナーマネージド環境での利用も計測可能
   - 顧客マネージド環境での利用も計測可能

3. **ビジネスインテリジェンス**
   - ソリューション別の AWS 消費量を可視化
   - マーケットプレイス販売とサービス利用の相関分析
   - ROI 計測と成長ポイントの特定

## 技術仕様

### タグ形式

| 項目 | 値 | 説明 |
|------|-----|------|
| タグキー | aws-apn-id | Partner Revenue Measurement 専用キー |
| タグ値形式 | pc:<product-code> | AWS Marketplace 商品コードを指定 |
| 適用対象 | すべての AWS リソース | EC2、RDS、Lambda など全リソースに適用可能 |

### 対応リージョン

AWS Partner Revenue Measurement は全商用リージョンで利用可能

## 設定方法

### 前提条件

1. AWS Marketplace での商品公開が完了
2. AWS Marketplace 商品のプロダクトコード確認
3. タグ適用権限が有る IAM ユーザー/ロール

### 手順

#### ステップ1: AWS Marketplace プロダクトコード確認

AWS Marketplace でのパートナーコンソールから、ソリューションのプロダクトコード (例: 1234abcd5678) を確認します。

#### ステップ2: タグポリシー作成

組織全体にタグを適用するため、AWS CloudFormation や Terraform でタグポリシーを定義します。

```json
{
  "tags": {
    "aws-apn-id": {
      "tag_key": {
        "arn:aws:ec2:*:*:instance/*": "pc:<your-product-code>"
      }
    }
  }
}
```

#### ステップ3: リソースにタグ適用

新規作成リソースまたは既存リソースに aws-apn-id タグを適用します。

```bash
aws ec2 create-tags \
  --resources i-1234567890abcdef0 \
  --tags Key=aws-apn-id,Value=pc:1234abcd5678
```

#### ステップ4: AWS コンソールで利用状況確認

AWS パートナーポータルで、タグ付きリソースからの AWS 消費量と収益インパクトを確認します。

## メリット

### ビジネス面

- **収益可視化**: ソリューション別の AWS 収益への貢献度を正確に把握できる
- **意思決定の最適化**: データ駆動型でマーケティングや開発投資を配分可能
- **成長機会の発見**: どのソリューションが最も AWS 利用を促進しているかを特定可能

### パートナーマネジメント面

- **顧客満足度**: 顧客が確認できるタグにより透明性が向上
- **信頼構築**: ソリューション価値を数字で実証可能

### 運用面

- **簡単な実装**: タグ付けという簡単な操作で利用開始可能
- **自動追跡**: タグ付けリソースからの利用は自動的に追跡

## 制限事項

- 新規リソースおよび既存リソースへのタグ適用は手動で実施
- AWS Marketplace プロダクトコードの事前確認が必要
- タグ適用忘れで計測漏れの可能性

## ユースケース

### ユースケース1: コンサルティングサービスの成果測定

**シナリオ**: AWS への移行支援を提供するコンサルティングパートナーが、顧客の AWS 支出増加への自社の貢献度を測定したい。

**実装**: 顧客環境のリソースに `aws-apn-id` タグを適用

**効果**: 提供したソリューション経由での AWS 消費増加を定量化でき、ROI を明確化可能

### ユースケース2: SaaS プロバイダーのビジネス評価

**シナリオ**: AWS 上で動作する独自の SaaS ソリューションを AWS Marketplace で提供しており、顧客利用による AWS サービス利用への影響を把握したい。

**実装**: SaaS インフラストラクチャのリソースに `aws-apn-id` タグを適用

**効果**: ソリューション利用に伴う AWS サービス利用量を把握でき、顧客ごとの収益貢献を測定可能

### ユースケース3: APN パートナーネットワークでのパフォーマンス評価

**シナリオ**: 複数の AWS パートナープログラムに参画しており、各プログラムごとの成果を測定したい。

**実装**: プログラムごとに異なるプロダクトコードで複数ソリューションをタグ付け

**効果**: プログラム別の AWS 収益への貢献度を比較分析でき、経営判断に活用可能

## まとめ

Partner Revenue Measurement により、AWS パートナーはソリューション提供による AWS 収益への実際の貢献度を可視化でき、より戦略的なビジネス意思決定が可能になります。シンプルなタグ付けで開始でき、顧客環境全体での利用追跡が可能なため、パートナーのビジネス成長と AWS との関係強化に貢献する重要なツールです。
