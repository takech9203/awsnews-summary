# Amazon Nova Act - UI ワークフロー自動化の一般提供開始

**リリース日**: 2025 年 12 月 2 日  
**サービス**: Amazon Nova Act  
**機能**: ブラウザベースタスク自動化 AI エージェント


## 概要

AWS は Amazon Nova Act の一般提供を発表しました。これは、開発者がブラウザベースのタスクを自動化する AI エージェントを構築するための新しいサービスです。フォーム入力、検索と抽出、ショッピングと予約、QA テストなどのタスクを、エンタープライズ環境で 90% 以上の信頼性で自動化できます。

Nova Act は、Web ブラウザを操作する AI エージェントを構築するための基盤を提供し、複雑な UI ワークフローを自然言語の指示で自動化することを可能にします。

**アップデート前の課題**

- Web 自動化は脆弱で、UI の変更により頻繁に壊れていた
- RPA (Robotic Process Automation) ツールは設定が複雑で、メンテナンスコストが高かった
- ブラウザベースのタスク自動化には専門的なスクリプト作成スキルが必要だった

**アップデート後の改善**

- AI がページ構造を理解し、UI 変更に対して堅牢な自動化を実現
- 自然言語でタスクを指示でき、専門的なスクリプト作成が不要に
- エンタープライズ環境で 90% 以上の信頼性を達成


## サービスアップデートの詳細

### 主要機能

1. **自然言語によるタスク指示**
   - 「このフォームに顧客情報を入力して送信」などの自然言語で指示

   - UI の軽微な変更に対して自動的に適応
   - エンタープライズ環境で 90% 以上の成功率

3. **多様なユースケース対応**
   - フォーム入力と送信
   - Web スクレイピングとデータ抽出
   - E コマースでの購入フロー
   - QA テストの自動化

4. **セキュリティとコンプライアンス**
   - 認証情報の安全な管理
   - 監査ログの記録
   - エンタープライズセキュリティ要件への対応

## 設定方法

### 前提条件

1. AWS アカウント
2. Amazon Nova Act へのアクセス権限
3. 自動化対象の Web アプリケーションへのアクセス

### 手順

#### ステップ 1: エージェントの作成

```python
import boto3

nova_act = boto3.client('nova-act', region_name='us-east-1')

# エージェントの作成
agent = nova_act.create_agent(
    name='form-automation-agent',
    description='フォーム入力を自動化するエージェント'
)
```

#### ステップ 2: タスクの定義と実行

```python
# タスクの実行
result = nova_act.execute_task(
    agentId=agent['agentId'],
    task={
        'instruction': 'https://example.com/contact にアクセスし、名前に「山田太郎」、メールに「yamada@example.com」を入力して送信ボタンをクリック',
        'timeout': 60
    }
)

print(f"タスク結果: {result['status']}")
```

## デメリット・制約事項

### 制限事項

- 高度にカスタマイズされた UI や非標準的な Web コンポーネントでは精度が低下する可能性
- リアルタイム性が求められるタスクには不向きな場合がある
- 一部の CAPTCHA や高度なボット対策には対応できない

### 考慮すべき点

- 自動化対象のサイトの利用規約を確認する必要がある
- 認証情報の安全な管理が重要


## ユースケース

### ユースケース 1: データ入力の自動化

**シナリオ**: 複数のシステムへの顧客データ入力を自動化

**実装例**:
```
Nova Act エージェントを使用:
- CSV ファイルから顧客データを読み込み
- 各システムのフォームに自動入力
- 入力結果を検証してレポート生成
```

**効果**: データ入力時間を 80% 削減、入力ミスを大幅に減少

### ユースケース 2: 競合価格モニタリング

**シナリオ**: 競合他社の Web サイトから価格情報を定期的に収集

**実装例**:
```
定期実行タスクを設定:
- 競合サイトにアクセス
- 製品価格を抽出
- データベースに保存してダッシュボードで可視化
```

**効果**: リアルタイムの競合分析が可能に

### ユースケース 3: E2E テストの自動化

**シナリオ**: Web アプリケーションのエンドツーエンドテストを自動化

**実装例**:
```
テストシナリオを自然言語で定義:
- ログインページでユーザー認証
- 商品を検索してカートに追加
- チェックアウトプロセスを完了
- 各ステップの結果を検証
```

**効果**: テストカバレッジを向上させながら、テスト実行時間を短縮
