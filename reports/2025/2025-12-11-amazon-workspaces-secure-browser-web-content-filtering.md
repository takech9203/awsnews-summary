# Amazon WorkSpaces Secure Browser - Web コンテンツフィルタリング

**リリース日**: 2025 年 12 月 11 日
**サービス**: Amazon WorkSpaces Secure Browser
**機能**: Web コンテンツフィルタリング

## 概要

Amazon WorkSpaces Secure Browser に Web コンテンツフィルタリング機能が追加されました。この包括的なセキュリティおよびコンプライアンス機能により、組織は Web コンテンツへのアクセスを制御・監視できます。管理者は詳細なアクセスポリシーを定義し、25 以上の事前定義カテゴリを使用して特定の URL やドメインカテゴリ全体をブロックできます。

既存の Chrome ポリシーによるドメイン制御は引き続きサポートされますが、Web コンテンツフィルタリングはカテゴリベースのフィルタリングと改善されたログ機能により、より包括的な Web アクセス制御を提供します。Session Logger との統合により、監視とコンプライアンスレポートも強化されています。

**アップデート前の課題**

- Chrome ポリシーによる URL 制御は限定的だった
- カテゴリベースのフィルタリングが利用できなかった
- コンプライアンスレポートのためのログ機能が不十分だった


**アップデート後の改善**

- 25 以上の事前定義カテゴリによるフィルタリング
- 詳細なアクセスポリシーの定義が可能
- Session Logger との統合によるコンプライアンスレポート強化

## サービスアップデートの詳細

### 主要機能

1. **カテゴリベースフィルタリング**
   - 25 以上の事前定義カテゴリ
   - ドメインカテゴリ全体のブロック
   - 特定 URL のブロック

2. **詳細なアクセスポリシー**
   - 組織のニーズに合わせたカスタマイズ
   - 例外設定の柔軟性
   - デフォルト拒否ポリシーのサポート

3. **コンプライアンス機能**
   - Session Logger との統合
   - 詳細なログと監視
   - コンプライアンスレポート

## 技術仕様

### フィルタリング機能

| 項目 | 詳細 |
|------|------|
| 事前定義カテゴリ | 25 以上 |
| URL フィルタリング | 特定 URL のブロック/許可 |
| ドメインフィルタリング | カテゴリ全体のブロック |
| ログ統合 | Session Logger |

### 既存機能との比較

| 機能 | Chrome ポリシー | Web コンテンツフィルタリング |
|------|----------------|---------------------------|
| URL ブロック | ○ | ○ |
| カテゴリフィルタリング | × | ○ |
| 詳細ログ | 限定的 | ○ |
| コンプライアンスレポート | × | ○ |

## 設定方法

### 前提条件

1. Amazon WorkSpaces Secure Browser ポータルの作成
2. 管理者権限
3. 対応リージョンでの利用

### 手順

#### ステップ 1: AWS コンソールにアクセス

AWS Management Console にログインし、WorkSpaces Secure Browser に移動します。

#### ステップ 2: Web コンテンツフィルタリングの有効化

ポータル設定で Web コンテンツフィルタリングを有効化します。既存の Chrome ポリシーの URL ブロックリスト/許可リストは自動的に移行されます。

#### ステップ 3: フィルタリングポリシーの設定

カテゴリベースのフィルタリングや特定 URL のブロック/許可を設定します。

## メリット

### ビジネス面

- **セキュリティ強化**: 不適切なコンテンツへのアクセスを防止
- **コンプライアンス対応**: 詳細なログとレポートで監査要件を満たす
- **一元管理**: 企業全体でポリシーを統一

### 技術面

- **包括的なフィルタリング**: カテゴリベースと URL ベースの両方をサポート
- **柔軟な設定**: ビジネスニーズに合わせたカスタマイズ
- **シームレスな移行**: 既存の Chrome ポリシーを自動移行

## デメリット・制約事項

### 制限事項

- 10 リージョンでのみ利用可能
- 事前定義カテゴリの変更は不可

### 考慮すべき点

- 過度に制限的なポリシーは業務効率に影響する可能性
- 例外設定の管理が必要

## ユースケース

### ユースケース 1: 高セキュリティ環境

**シナリオ**: 金融機関で、従業員のインターネットアクセスを厳格に制御したい

**効果**: デフォルト拒否ポリシーを実装し、業務に必要なサイトのみを許可

### ユースケース 2: コンプライアンス監査

**シナリオ**: 規制要件に基づいて Web アクセスの監査証跡が必要

**効果**: Session Logger との統合により、詳細なアクセスログを取得

### ユースケース 3: リモートワークセキュリティ

**シナリオ**: BYOD 環境でのセキュアなブラウジングを提供したい

**効果**: カテゴリベースのフィルタリングで不適切なコンテンツをブロック

## 料金

Web コンテンツフィルタリング機能は追加料金なしで利用できます。WorkSpaces Secure Browser の標準料金のみが適用されます。
