# Amazon Bedrock AgentCore Runtime - 直接コードデプロイサポート

**リリース日**: 2025 年 11 月 10 日  
**サービス**: Amazon Bedrock  
**機能**: AgentCore Runtime 直接コードデプロイ


## 概要

Amazon Bedrock AgentCore Runtime は、AI エージェントのデプロイ方法として、コンテナベースのデプロイに加えて直接コードアップロードをサポートしました。迅速なプロトタイピングと反復のための直接コード (zip ファイル) アップロード、または複雑なユースケース向けのカスタム構成が必要なコンテナベースのオプションを選択できます。

AgentCore Runtime は、エージェントとツールを大規模に実行するためのサーバーレスフレームワークとモデル非依存のランタイムを提供します。

**アップデート前の課題**

- AI エージェントのデプロイにはコンテナイメージの作成が必須で、プロトタイピングに時間がかかった
- 簡単なテストや反復的な開発でも、コンテナビルドのオーバーヘッドが発生していた
- 開発者がコンテナ技術に精通している必要があった

**アップデート後の改善**

- zip ファイルによる直接コードアップロードで、数分でエージェントをデプロイできるようになった
- ドラッグアンドドロップ機能により、迅速なプロトタイピングと反復が可能になった
- ユースケースに応じて、直接コードアップロードとコンテナベースデプロイを選択できるようになった


## サービスアップデートの詳細

### 主要機能

1. **直接コードアップロード**
   - zip ファイルによるコードアップロード
   - ドラッグアンドドロップ機能
   - 迅速なプロトタイピングに最適

2. **コンテナベースデプロイ**
   - カスタム構成が必要な複雑なユースケース向け
   - 既存のコンテナワークフローとの統合
   - 本番環境向けの柔軟性

3. **サーバーレスフレームワーク**
   - インフラストラクチャ管理不要
   - 自動スケーリング
   - モデル非依存のランタイム


## 技術仕様

### デプロイオプション比較

| 項目 | 直接コードアップロード | コンテナベース |
|------|---------------------|--------------|
| 用途 | プロトタイピング、迅速な反復 | 本番環境、複雑な構成 |
| セットアップ時間 | 短い | 中程度 |
| カスタマイズ性 | 限定的 | 高い |
| デプロイ方法 | zip ファイル | コンテナイメージ |

### サポートされるランタイム

| ランタイム | バージョン |
|----------|----------|
| Python | 3.9, 3.10, 3.11, 3.12 |
| Node.js | 18.x, 20.x |


## 設定方法

### 前提条件

1. Amazon Bedrock へのアクセス権限
2. AgentCore Runtime が有効化されていること
3. 適切な IAM ロール

### 手順

#### ステップ 1: コードの準備

```python
# agent.py
def handler(event, context):
    # エージェントロジック
    return {
        "statusCode": 200,
        "body": "Agent response"
    }
```

#### ステップ 2: zip ファイルの作成

```bash
zip -r agent.zip agent.py requirements.txt
```

#### ステップ 3: AgentCore Runtime へのデプロイ

AWS コンソールまたは CLI を使用して zip ファイルをアップロードします。

## デメリット・制約事項

### 制限事項

- 直接コードアップロードにはファイルサイズ制限がある
- 一部の複雑な依存関係はコンテナベースが推奨

### 考慮すべき点

- プロトタイプから本番への移行計画
- 適切なデプロイ方法の選択
