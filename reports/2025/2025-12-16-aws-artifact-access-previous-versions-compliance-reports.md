# AWS Artifact - 過去バージョンのコンプライアンスレポートへのアクセス

**リリース日**: 2025 年 12 月 16 日
**サービス**: AWS Artifact
**機能**: 過去バージョンのコンプライアンスレポートへのセルフサービスアクセス

## 概要

AWS Artifact で過去バージョンの AWS コンプライアンスレポートに直接アクセスできるようになりました。これまでは過去のレポートを取得するために AWS サポートやアカウント担当者への連絡が必要でしたが、今回のアップデートによりセルフサービスで取得可能になりました。

この機能は、監査やベンダー評価において過去のコンプライアンス証跡が必要な場合に特に有用です。SOC、ISO、C5 などのコンプライアンスレポートの過去バージョンを AWS Artifact コンソールから直接ダウンロードできます。

**アップデート前の課題**

- 過去のコンプライアンスレポートを取得するには AWS サポートへの連絡が必要だった
- 監査対応時に過去のレポート取得に時間がかかっていた
- ベンダー評価で過去の証跡を迅速に提供することが困難だった

**アップデート後の改善**

- AWS Artifact コンソールから過去バージョンのレポートに直接アクセス可能
- セルフサービスにより、即座にレポートをダウンロード
- 監査やベンダー評価への迅速な対応が可能に

## サービスアップデートの詳細

### 主要機能

1. **セルフサービスアクセス**
   - AWS Artifact コンソールから過去バージョンのレポートを直接ダウンロード
   - AWS サポートへの連絡が不要に
   - 即座にレポートを取得可能

2. **対応するコンプライアンスプログラム**
   - SOC (Service Organization Control) レポート
   - ISO 認証レポート
   - C5 (Cloud Computing Compliance Controls Catalogue) レポート
   - その他の AWS コンプライアンスレポート

3. **IAM 権限による制御**
   - `artifact:ListReportVersions` 権限で過去バージョンへのアクセスを制御
   - AWS マネージドポリシー `AWSArtifactReportsReadOnlyAccess` に含まれる

## 技術仕様

### 必要な IAM 権限

| 権限 | 説明 |
|------|------|
| `artifact:ListReportVersions` | 過去バージョンのレポート一覧を表示 |
| `artifact:GetReport` | レポートをダウンロード |

### IAM ポリシー例

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "artifact:ListReportVersions",
                "artifact:GetReport",
                "artifact:ListReports"
            ],
            "Resource": "*"
        }
    ]
}
```

過去バージョンのレポートにアクセスするために必要な IAM 権限を設定します。

### 利用可能なレポートタイプ

| レポートタイプ | 説明 | 過去バージョンの可用性 |
|---------------|------|----------------------|
| SOC 1 | 財務報告に関連する内部統制 | 複数年分 |
| SOC 2 | セキュリティ、可用性、処理の完全性、機密性、プライバシー | 複数年分 |
| SOC 3 | SOC 2 の公開版 | 複数年分 |
| ISO 27001 | 情報セキュリティマネジメント | 複数年分 |
| ISO 27017 | クラウドセキュリティ | 複数年分 |
| ISO 27018 | クラウドプライバシー | 複数年分 |
| C5 | ドイツ連邦情報セキュリティ庁の基準 | 限定的 |

## 設定方法

### 前提条件

1. AWS アカウント
2. `artifact:ListReportVersions` 権限を持つ IAM ユーザーまたはロール
3. AWS Artifact コンソールへのアクセス

### 手順

#### ステップ 1: IAM 権限の確認

```bash
# 現在のユーザーの権限を確認
aws iam get-user-policy --user-name <username> --policy-name <policy-name>

# または、マネージドポリシーをアタッチ
aws iam attach-user-policy \
    --user-name <username> \
    --policy-arn arn:aws:iam::aws:policy/AWSArtifactReportsReadOnlyAccess
```

`AWSArtifactReportsReadOnlyAccess` マネージドポリシーには必要な権限が含まれています。

#### ステップ 2: AWS Artifact コンソールでのアクセス

1. AWS マネジメントコンソールにログイン
2. AWS Artifact コンソールを開く
3. 「Reports」ページに移動
4. 目的のレポートを選択
5. 「Versions」タブで過去バージョンを確認
6. ダウンロードするバージョンを選択

#### ステップ 3: レポートのダウンロード

コンソールから直接レポートをダウンロードします。NDA (秘密保持契約) への同意が必要な場合があります。

## メリット

### ビジネス面

- **監査対応の迅速化**: 過去のコンプライアンス証跡を即座に取得し、監査対応時間を短縮
- **ベンダー評価の効率化**: 顧客やパートナーからの要求に迅速に対応
- **コンプライアンス管理の簡素化**: セルフサービスにより、コンプライアンスドキュメントの管理が容易に

### 技術面

- **即時アクセス**: AWS サポートへの連絡なしで即座にレポートを取得
- **IAM による制御**: 組織内でのアクセス権限を細かく制御可能
- **一元管理**: AWS Artifact コンソールで現在および過去のレポートを一元管理

## デメリット・制約事項

### 制限事項

- 利用可能なリージョンが限定されている (US East (N. Virginia) および AWS GovCloud (US-West))
- 過去バージョンの可用性はコンプライアンスプログラムによって異なる
- 一部のレポートは過去バージョンが限定的

### 考慮すべき点

- 過去バージョンへのアクセスには適切な IAM 権限が必要
- NDA への同意が必要なレポートがある

## ユースケース

### ユースケース 1: 年次監査対応

**シナリオ**: 外部監査人から過去 3 年分の SOC 2 レポートの提出を求められた

**実装例**:
1. AWS Artifact コンソールにアクセス
2. SOC 2 レポートを選択
3. 過去 3 年分のバージョンをダウンロード
4. 監査人に提出

**効果**: AWS サポートへの連絡なしで即座にレポートを取得し、監査対応時間を大幅に短縮

### ユースケース 2: ベンダー評価への対応

**シナリオ**: 新規顧客から AWS のセキュリティ認証の履歴を求められた

**実装例**:
1. AWS Artifact コンソールにアクセス
2. ISO 27001、ISO 27017、ISO 27018 レポートを選択
3. 現在および過去のバージョンをダウンロード
4. 顧客に提供

**効果**: 顧客の要求に迅速に対応し、ビジネス機会を逃さない

### ユースケース 3: コンプライアンス履歴の管理

**シナリオ**: 社内のコンプライアンスチームが AWS のコンプライアンス状況の履歴を管理

**実装例**:
1. 定期的に AWS Artifact から最新レポートをダウンロード
2. 過去バージョンと比較して変更点を確認
3. 社内のコンプライアンス管理システムに記録

**効果**: AWS のコンプライアンス状況の変化を継続的に追跡

## まとめ

AWS Artifact の過去バージョンコンプライアンスレポートへのセルフサービスアクセス機能により、監査やベンダー評価への対応が大幅に効率化されます。AWS サポートへの連絡なしで即座に過去のレポートを取得できるため、コンプライアンス管理の負担が軽減されます。特に、定期的な監査対応や顧客からのセキュリティ評価要求に迅速に対応する必要がある組織にとって有用な機能です。
