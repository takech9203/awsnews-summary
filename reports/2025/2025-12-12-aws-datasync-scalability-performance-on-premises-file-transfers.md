# AWS DataSync - Enhanced モードのオンプレミスファイル転送サポート

**リリース日**: 2025 年 12 月 12 日  
**サービス**: AWS DataSync  
**機能**: Enhanced モードによるオンプレミスファイルサーバーと Amazon S3 間のデータ転送


## 概要

AWS DataSync Enhanced モードは、オンプレミスファイルサーバーと Amazon S3 間のデータ転送をサポートするようになりました。この機能により、事実上無制限のファイル数を持つデータセットを、DataSync Basic モードよりも高いパフォーマンスレベルで転送できます。

Enhanced モードは並列処理を使用して、あらゆるサイズのデータセットに対してより高いパフォーマンスとスケーラビリティを提供し、ファイル数の制限を排除し、より良い監視と管理のための詳細な転送メトリクスを提供します。

**アップデート前の課題**

- 以前は DataSync Basic モードでオンプレミスから S3 への転送を行う際、ファイル数に制限があった
- 以前は大規模データセットの転送パフォーマンスが限定的で、生成 AI トレーニングデータなどの移行に時間がかかっていた
- 以前は転送の詳細なメトリクスが限定的で、進捗監視や問題分析が困難だった

**アップデート後の改善**

- 今回のアップデートにより、Enhanced モードで事実上無制限のファイル数を持つデータセットを転送できるようになった
- 今回のアップデートにより、並列処理による高パフォーマンス転送で大規模データセットの移行時間を短縮できるようになった
- 今回のアップデートにより、詳細な転送メトリクスで進捗監視とパフォーマンス分析が容易になった


## サービスアップデートの詳細

### 主要機能

1. **並列処理**
   - 高パフォーマンスの転送
   - あらゆるサイズのデータセットに対応
   - ファイル数制限の排除

2. **詳細な転送メトリクス**
   - 転送進捗の監視
   - パフォーマンス分析
   - 管理の効率化

3. **サポートされるプロトコル**
   - NFS (Network File System)
   - SMB (Server Message Block)


## 技術仕様

### サポートされる転送パス

| ソース | デスティネーション |
|--------|------------------|
| オンプレミス NFS サーバー | Amazon S3 |
| オンプレミス SMB サーバー | Amazon S3 |

### Enhanced モードの特徴

| 特徴 | 説明 |
|------|------|
| 並列処理 | 複数のファイルを同時に転送 |
| スケーラビリティ | 事実上無制限のファイル数 |
| メトリクス | 詳細な転送統計 |


## 設定方法

### 前提条件

1. AWS DataSync エージェントがオンプレミスにデプロイ済み
2. NFS または SMB ファイルサーバーへのアクセス
3. Amazon S3 バケットが設定済み

### 手順

#### ステップ 1: DataSync エージェントの確認

オンプレミス環境に DataSync エージェントがデプロイされていることを確認します。

#### ステップ 2: ロケーションの作成

ソース (NFS/SMB) とデスティネーション (S3) のロケーションを作成します。

#### ステップ 3: タスクの作成

Enhanced モードを選択してデータ転送タスクを作成します。


## メリット

### ビジネス面

- **生成 AI ワークロードの加速**: トレーニングデータセットを AWS に迅速に移動
- **データレイク分析**: オンプレミスデータをクラウドベースのパイプラインと同期
- **大規模マイグレーション**: アーカイブとクラウドモダナイゼーションの推進

### 技術面

- **高パフォーマンス**: 並列処理による転送速度向上
- **スケーラビリティ**: ファイル数制限なし
- **詳細な監視**: 転送メトリクスによる可視性向上


## デメリット・制約事項

### 制限事項

- オンプレミス環境に DataSync エージェントが必要
- ネットワーク帯域幅に依存

### 考慮すべき点

- 大量データ転送時のネットワークコストを事前に見積もること
- エージェントのリソース要件を確認すること


## ユースケース

### ユースケース 1: 生成 AI トレーニングデータの移行

**シナリオ**: 大量のトレーニングデータセットをオンプレミスから AWS に移動したい

**効果**: Enhanced モードにより、大規模データセットを高速に転送し、AI ワークロードを加速

### ユースケース 2: データレイクへのデータ同期

**シナリオ**: オンプレミスのファイルサーバーデータを S3 データレイクと同期したい

**効果**: 継続的なデータ同期により、クラウドベースの分析パイプラインを強化

### ユースケース 3: クラウドモダナイゼーション

**シナリオ**: オンプレミスのアーカイブデータを AWS に移行したい

**効果**: 大規模マイグレーションを効率的に実行し、クラウドモダナイゼーションを推進
