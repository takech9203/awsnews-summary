# Amazon Bedrock AgentCore - Policy と Evaluations のプレビュー開始

**リリース日**: 2025 年 12 月 2 日  
**サービス**: Amazon Bedrock AgentCore  
**機能**: Policy (プレビュー)、Evaluations (プレビュー)、Memory/Runtime/Identity の機能強化


## 概要

Amazon Bedrock AgentCore に、Policy (プレビュー) と Evaluations (プレビュー) を含む新機能が追加されました。これらの機能により、チームはエージェントのデプロイメントを組織全体で自信を持ってスケールするために必要なコントロールと品質保証を得ることができます。

Policy in AgentCore は AgentCore Gateway と統合し、すべてのツール呼び出しをリアルタイムでインターセプトして、エージェントが定義された境界内に留まることを保証します。自然言語でポリシーを作成でき、AWS のオープンソースポリシー言語である Cedar に自動変換されます。

AgentCore Evaluations は、開発者が実際の動作に基づいてエージェントのパフォーマンスをテストし、継続的に監視することを支援します。13 の組み込みエバリュエーターを使用して、有用性、ツール選択、精度などの一般的な品質ディメンションを評価できます。

**アップデート前の課題**

- AI エージェントのアクションに対する細かい制御が困難で、意図しない動作を防ぐのが難しかった
- エージェントの品質を継続的に監視・評価するためのインフラストラクチャを独自に構築する必要があった
- エージェントが過去の経験から学習し、より人間らしいインタラクションを実現することが困難だった

**アップデート後の改善**

- Policy により、自然言語でポリシーを定義し、エージェントのアクションをリアルタイムで制御可能に
- Evaluations により、13 の組み込みエバリュエーターで品質を継続的に監視、カスタムスコアリングも可能
- Episodic Memory により、エージェントが経験から学習し、より人間らしいインタラクションを実現


## サービスアップデートの詳細

### 主要機能

1. **Policy in AgentCore (プレビュー)**
   - AgentCore Gateway と統合し、ツール呼び出しをリアルタイムでインターセプト
   - 自然言語でポリシーを作成し、Cedar に自動変換
   - 開発、コンプライアンス、セキュリティチームがカスタムコードなしでルールを設定・監査可能

2. **AgentCore Evaluations (プレビュー)**
   - 実際の動作に基づいてエージェントパフォーマンスをテスト・監視
   - 13 の組み込みエバリュエーター（有用性、ツール選択、精度など）
   - カスタムモデルベースのスコアリングシステムを作成可能
   - Amazon CloudWatch を活用した統合ダッシュボード

3. **AgentCore Memory の強化**
   - Episodic Memory: エージェントが経験から学習し、時間とともに知識を構築
   - より人間らしいインタラクションを実現

4. **AgentCore Runtime の強化**
   - 双方向ストリーミング: エージェントが同時にリッスンと応答を実行
   - 会話中の中断やコンテキスト変更に対応
   - 音声エージェントのユースケースを実現

5. **AgentCore Identity の強化**
   - カスタムクレームによる認証ルールの強化
   - マルチテナント環境でのシームレスな ID プロバイダー統合


## 技術仕様

### Policy の仕組み

| 項目 | 詳細 |
|------|------|
| 統合先 | AgentCore Gateway |
| ポリシー言語 | Cedar（自然言語から自動変換） |
| 動作 | ツール呼び出しをリアルタイムでインターセプト |
| 対象 | 開発、コンプライアンス、セキュリティチーム |

### Evaluations の組み込みエバリュエーター

| エバリュエーター | 説明 |
|-----------------|------|
| Helpfulness | 応答の有用性を評価 |
| Tools Selection | 適切なツール選択を評価 |
| Accuracy | 応答の正確性を評価 |
| その他 10 種類 | 一般的な品質ディメンションをカバー |


## メリット

### ビジネス面

- **コンプライアンス対応**: 自然言語でポリシーを定義し、監査可能なルールを設定
- **品質保証**: 継続的な監視により、問題を顧客影響前に検出
- **スケーラビリティ**: プロトタイプから本番環境へのエージェント展開を加速

### 技術面

- **開発効率**: カスタムコードなしでポリシーと評価インフラを構築
- **統合監視**: CloudWatch ダッシュボードで品質メトリクスを一元管理
- **柔軟性**: カスタムスコアリングシステムで独自の評価基準を定義可能


## デメリット・制約事項

### 制限事項

- Evaluations は 4 リージョンのみでプレビュー提供（US East (N. Virginia)、US West (Oregon)、Asia Pacific (Sydney)、Europe (Frankfurt)）
- Policy は AgentCore が利用可能な全リージョンでプレビュー提供
- プレビュー段階のため、本番環境での使用には注意が必要

### 考慮すべき点

- Cedar ポリシー言語の学習が必要な場合がある
- 評価基準のカスタマイズには追加の設計が必要


## ユースケース

### ユースケース 1: コンプライアンス要件への対応

**シナリオ**: 金融機関で AI エージェントが顧客データにアクセスする際のコンプライアンス要件

**実装例**:
```
Policy: "エージェントは顧客の個人情報を外部サービスに送信してはならない"
→ Cedar に自動変換され、ツール呼び出しをリアルタイムで検証
```

**効果**: コンプライアンス違反を自動的に防止し、監査証跡を提供

### ユースケース 2: エージェント品質の継続的監視

**シナリオ**: カスタマーサポートエージェントの応答品質を継続的に監視

**実装例**:
```
Evaluations 設定:
- Helpfulness エバリュエーター: 有用性スコア 80% 以上を目標
- Accuracy エバリュエーター: 正確性スコア 90% 以上を目標
- カスタムスコアリング: 顧客満足度予測モデル
```

**効果**: 品質低下を早期に検出し、顧客影響前に対処

### ユースケース 3: 音声エージェントの構築

**シナリオ**: コールセンター向けの音声 AI エージェント

**実装例**:
```
AgentCore Runtime の双方向ストリーミングを活用:
- ユーザーの発話をリアルタイムで処理
- 中断やコンテキスト変更に対応
- 自然な会話フローを実現
```

**効果**: より人間らしい音声インタラクションを実現
