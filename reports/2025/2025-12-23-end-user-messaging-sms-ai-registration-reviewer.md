# AWS End User Messaging SMS - 生成 AI 登録レビューアー (プレビュー)

**リリース日**: 2025年12月23日
**サービス**: AWS End User Messaging
**機能**: 生成 AI 登録レビューアー (プレビュー)

## 概要

AWS End User Messaging は、電話番号登録を確認するための生成 AI 登録レビューアーをプレビューとして発表しました。この機能により、モバイルキャリアへの登録を初回で正しく提出できるようになります。登録レビューアー (プレビュー) を使用すると、AWS がメッセージサンプル、オプトイン説明、ユースケース、ヘルプおよび停止メッセージなどをチェックし、正確で完全な登録を提出するためのフィードバックを提供します。

AWS End User Messaging は、通信の安全性、セキュリティ、結果を損なうことなく、スケーラブルでコスト効率の高いメッセージングインフラストラクチャを開発者に提供します。開発者は、サインアップ時のワンタイムパスコード (OTP)、アカウント更新、予約リマインダー、配送通知、プロモーションなどのユースケースをサポートするためにメッセージングを統合できます。

**アップデート前の課題**

- 電話番号登録がキャリアに拒否されることが多かった
- 登録フォームの不備を事前に検出できなかった
- 登録の再提出に時間とコストがかかっていた

**アップデート後の改善**

- 生成 AI が登録フォームを事前にレビューし、問題点を指摘
- メッセージサンプル、オプトイン説明、ユースケースなどを自動チェック
- 初回で正しい登録を提出できる確率が向上

## サービスアップデートの詳細

### 主要機能

1. **登録フォームの AI レビュー**
   - メッセージサンプルの検証
   - オプトイン説明の確認
   - ユースケースの適切性チェック
   - ヘルプ/停止メッセージの検証

2. **フィードバック提供**
   - 問題点の具体的な指摘
   - 改善提案の提示
   - キャリア要件との整合性確認

3. **プレビュー機能**
   - 提出前の事前確認
   - 修正機会の提供
   - 登録成功率の向上

## 技術仕様

### チェック項目

| 項目 | 説明 |
|------|------|
| メッセージサンプル | 送信予定のメッセージ例の適切性 |
| オプトイン説明 | ユーザー同意取得方法の説明 |
| ユースケース | メッセージング目的の明確性 |
| ヘルプメッセージ | HELP キーワードへの応答内容 |
| 停止メッセージ | STOP キーワードへの応答内容 |
| ブランド情報 | 送信者情報の正確性 |

### サポートされる登録タイプ

| 登録タイプ | サポート状況 |
|-----------|-------------|
| 10DLC (10 桁ロングコード) | ✓ |
| ショートコード | ✓ |
| トールフリー番号 | ✓ |

## 設定方法

### 前提条件

1. AWS End User Messaging が有効化されていること
2. 電話番号登録の準備が完了していること
3. 適切な IAM 権限

### 手順

#### ステップ 1: AWS End User Messaging コンソールへのアクセス

AWS マネジメントコンソールから End User Messaging にアクセスします。

#### ステップ 2: 登録フォームの作成

電話番号登録に必要な情報を入力します:

1. **ブランド情報**
   - 会社名
   - ウェブサイト URL
   - 連絡先情報

2. **キャンペーン情報**
   - ユースケースの説明
   - メッセージサンプル
   - 送信頻度

3. **コンプライアンス情報**
   - オプトイン方法の説明
   - ヘルプメッセージ
   - 停止メッセージ

#### ステップ 3: AI レビューの実行

登録フォームを保存する前に、AI レビューを実行します:

1. 「AI でレビュー」ボタンをクリック
2. AI がフォームを分析
3. フィードバックを確認

#### ステップ 4: フィードバックに基づく修正

AI からのフィードバックに基づいて登録フォームを修正します:

```
例: AI フィードバック
- メッセージサンプルにブランド名が含まれていません
- オプトイン説明が不十分です。具体的な同意取得方法を記載してください
- STOP メッセージの応答が標準形式に準拠していません
```

#### ステップ 5: 登録の提出

修正完了後、キャリアに登録を提出します。

## メリット

### ビジネス面

- **時間短縮**: 登録の再提出回数を削減
- **コスト削減**: 拒否による遅延コストを回避
- **成功率向上**: 初回承認率の向上

### 技術面

- **自動チェック**: 手動レビューの負担を軽減
- **一貫性**: キャリア要件との整合性を自動確認
- **学習機能**: 最新のキャリア要件に対応

## デメリット・制約事項

### 制限事項

- プレビュー機能のため、一部の機能が制限される可能性
- AI の判断が 100% 正確とは限らない
- 最終的な承認はキャリアの判断による

### 考慮すべき点

- AI フィードバックは参考情報として活用
- キャリア固有の要件は別途確認が必要
- プレビュー期間中は機能が変更される可能性

## ユースケース

### ユースケース 1: OTP (ワンタイムパスコード) サービス

**シナリオ**: 認証用 OTP を送信するための電話番号登録

**実装例**:
- ユースケース: 「ユーザー認証のためのワンタイムパスコード送信」
- メッセージサンプル: 「[ブランド名] 認証コード: 123456。このコードは 10 分間有効です。」
- AI レビューで OTP 形式の適切性を確認

**効果**: OTP サービスに適した登録フォームを初回で承認

### ユースケース 2: 予約リマインダー

**シナリオ**: 医療機関や美容院の予約リマインダー送信

**実装例**:
- ユースケース: 「予約確認とリマインダーの送信」
- メッセージサンプル: 「[クリニック名] 明日 14:00 のご予約をお忘れなく。変更は 03-xxxx-xxxx まで。」
- オプトイン: 「予約時にSMS通知への同意を取得」

**効果**: 医療機関向けの適切な登録フォームを作成

### ユースケース 3: プロモーション通知

**シナリオ**: EC サイトのセール情報やクーポン配信

**実装例**:
- ユースケース: 「マーケティングプロモーションとセール通知」
- メッセージサンプル: 「[ストア名] 週末限定 20% OFF! コード: WEEKEND20。詳細: example.com」
- オプトイン: 「会員登録時にマーケティングメッセージへの同意を取得」

**効果**: マーケティング用途に適した登録を承認

※ 実際の料金はリージョン、送信先、メッセージタイプにより異なります。
