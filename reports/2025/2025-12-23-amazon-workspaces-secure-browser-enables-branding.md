# Amazon WorkSpaces Secure Browser - ブランディングカスタマイズ

**リリース日**: 2025年12月23日
**サービス**: Amazon WorkSpaces Secure Browser
**機能**: ブランディングカスタマイズ

## 概要

Amazon WorkSpaces Secure Browser は、ブランディングカスタマイズ機能をサポートしました。この機能により、組織のビジュアルアイデンティティに合わせた一貫性のあるブランド体験を作成できます。エンドユーザーに表示されるサインイン画面とセッション読み込み画面をカスタマイズし、すべてのユーザータッチポイントでブランドの一貫性を維持できます。

ファビコン、ロゴ、壁紙のアップロード、カラーテーマの選択、ウェルカムメッセージやブラウザタブタイトルのカスタマイズが可能です。サービスがサポートする 11 言語すべてでテキストフィールドをカスタマイズできます。また、「お問い合わせ」リンクを組織のサポートページにリダイレクトしたり、セッション開始前にユーザーが確認する必要がある利用規約ページを追加することもできます。

**アップデート前の課題**

- サインイン画面やセッション読み込み画面をカスタマイズできなかった
- 組織のブランドアイデンティティを反映した体験を提供できなかった
- 利用規約の同意確認を別途実装する必要があった

**アップデート後の改善**

- ロゴ、ファビコン、壁紙、カラーテーマを自由にカスタマイズ可能
- 11 言語でウェルカムメッセージやテキストをカスタマイズ可能
- 利用規約ページの追加とユーザー同意確認が組み込み機能として利用可能

## サービスアップデートの詳細

### 主要機能

1. **ビジュアル要素のカスタマイズ**
   - ファビコンのアップロード
   - 組織ロゴのアップロード
   - 壁紙画像の設定
   - カラーテーマの選択

2. **テキストコンテンツのカスタマイズ**
   - ウェルカムメッセージの編集
   - ブラウザタブタイトルの変更
   - 11 言語でのテキストフィールドカスタマイズ
   - 「お問い合わせ」リンクのカスタムURL設定

3. **利用規約機能**
   - 利用規約ページの追加
   - セッション開始前のユーザー同意確認
   - カスタム利用規約コンテンツの設定

4. **アクセシビリティ対応**
   - WCAG AA アクセシビリティ要件に準拠
   - コントラスト要件を満たす設計

## 技術仕様

### カスタマイズ可能な要素

| 要素 | 説明 | 形式/制限 |
|------|------|----------|
| ファビコン | ブラウザタブに表示されるアイコン | ICO/PNG 形式 |
| ロゴ | サインイン画面に表示される組織ロゴ | PNG/JPG 形式 |
| 壁紙 | 背景画像 | PNG/JPG 形式 |
| カラーテーマ | UI のカラースキーム | プリセットから選択 |
| ウェルカムメッセージ | サインイン画面のメッセージ | テキスト (11 言語対応) |
| タブタイトル | ブラウザタブに表示されるタイトル | テキスト |

### サポート言語

11 言語でテキストコンテンツをカスタマイズ可能:
- 英語、日本語、ドイツ語、フランス語、スペイン語、イタリア語、ポルトガル語、韓国語、中国語 (簡体字/繁体字)、その他

## 設定方法

### 前提条件

1. Amazon WorkSpaces Secure Browser ポータルが作成済みであること
2. 適切な IAM 権限を持つ AWS アカウント
3. カスタマイズ用の画像ファイル (ロゴ、ファビコン、壁紙)

### 手順

#### ステップ 1: WorkSpaces Secure Browser コンソールへのアクセス

AWS マネジメントコンソールから WorkSpaces Secure Browser にアクセスし、対象のポータルを選択します。

#### ステップ 2: ブランディング設定の構成

ポータル設定からブランディングカスタマイズセクションに移動し、以下を設定します:

1. **ロゴのアップロード**
   - 組織のロゴ画像をアップロード
   - 推奨サイズと形式に従って準備

2. **ファビコンの設定**
   - ブラウザタブ用のアイコンをアップロード

3. **壁紙の設定**
   - 背景画像をアップロード
   - または単色の背景色を選択

4. **カラーテーマの選択**
   - 組織のブランドカラーに合わせたテーマを選択

#### ステップ 3: テキストコンテンツのカスタマイズ

1. **ウェルカムメッセージの編集**
   - 各言語でカスタムメッセージを入力

2. **タブタイトルの設定**
   - ブラウザタブに表示されるタイトルを設定

3. **お問い合わせリンクの設定**
   - 組織のサポートページ URL を指定

#### ステップ 4: 利用規約の設定 (オプション)

1. 利用規約ページを有効化
2. 利用規約コンテンツを入力
3. ユーザー同意確認を有効化

## メリット

### ビジネス面

- **ブランド一貫性**: 組織のビジュアルアイデンティティを維持
- **プロフェッショナルな印象**: カスタマイズされた体験でユーザーの信頼を向上
- **コンプライアンス対応**: 利用規約の同意確認を組み込み機能で実現

### 技術面

- **追加コストなし**: 既存の WorkSpaces Secure Browser 料金に含まれる
- **アクセシビリティ準拠**: WCAG AA 要件を満たす設計
- **多言語対応**: 11 言語でのカスタマイズが可能

## デメリット・制約事項

### 制限事項

- 一部のリージョンでは利用不可
- 画像ファイルのサイズと形式に制限がある
- カラーテーマはプリセットから選択 (完全なカスタムカラーは不可)

### 考慮すべき点

- ブランディング変更は全ユーザーに即座に反映される
- アクセシビリティ要件を満たすコントラスト比の確保が必要
- 多言語環境では各言語でのテキスト準備が必要

## ユースケース

### ユースケース 1: 企業向けセキュアブラウジング環境

**シナリオ**: 大企業が従業員向けにセキュアなブラウジング環境を提供し、企業ブランドを反映

**実装例**:
- 企業ロゴとブランドカラーを設定
- 社内サポートページへのリンクを追加
- IT 利用ポリシーの同意確認を設定

**効果**: 従業員に一貫したブランド体験を提供しながら、セキュリティポリシーの遵守を確保

### ユースケース 2: マネージドサービスプロバイダー (MSP)

**シナリオ**: MSP が顧客ごとにブランド化されたセキュアブラウジングサービスを提供

**実装例**:
- 顧客ごとに異なるポータルを作成
- 各顧客のロゴとブランドカラーを設定
- 顧客固有のサポート連絡先を設定

**効果**: ホワイトラベルサービスとして顧客に提供可能

### ユースケース 3: 規制産業向けコンプライアンス対応

**シナリオ**: 金融機関や医療機関が規制要件を満たすセキュアブラウジング環境を構築

**実装例**:
- 利用規約ページで法的免責事項を表示
- セッション開始前の同意確認を必須化
- 監査証跡のための同意記録

**効果**: 規制要件への準拠を証明可能な形で実装

## 料金

ブランディングカスタマイズ機能は追加料金なしで利用可能です。標準の WorkSpaces Secure Browser 従量課金制料金が適用されます。

### 料金例

| 使用量 | 月額料金（概算） |
|--------|------------------|
| 100 ユーザー × 20 時間/月 | 約 $200〜300 USD |
| 500 ユーザー × 40 時間/月 | 約 $2,000〜3,000 USD |

※ 実際の料金はリージョンや使用状況により異なります。
